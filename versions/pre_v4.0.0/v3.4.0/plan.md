# Favnir v3.4.0 Implementation Plan

## Overview

v3.4.0 は `fav infer` コマンドを追加する。
新しい VM プリミティブや型システム拡張は**ない** —
実装はほぼ `driver.rs` と `main.rs` の Rust コードで完結する。

既存の CSV パース（`csv` クレート）と SQLite 接続（`rusqlite`）を再利用する。
PostgreSQL 対応は `postgres_integration` feature フラグでゲート（v3.3.0 と同様）。

Total phases: 7

## Phase 2: 型推論コア (`driver.rs`)

**Goal**: CSV / DB 推論に共通する内部型と推論ロジックを実装する。

### 2-A: 内部型

```rust
#[derive(Debug, Clone, PartialEq)]
enum InferredType {
    Int,
    Float,
    Bool,
    FavString,                    // String (キーワード衝突避け)
    Option(Box<InferredType>),
}

#[derive(Debug, Clone)]
struct InferredField {
    name: String,
    ty:   InferredType,
}

#[derive(Debug, Clone)]
struct InferredTypeDef {
    name:   String,
    fields: Vec<InferredField>,
    source: String,   // コメント用 ("data.csv" / "--db ... users")
}
```

### 2-B: 型推論ヘルパー

```rust
/// サンプル値群からカラムの型を推論する
fn infer_type_from_values(values: &[&str]) -> InferredType {
    // 全値が空 → Option<String>
    // 非空値が存在する場合:
    //   全て bool パターン → Bool
    //   全て int パターン → Int
    //   全て float パターン → Float
    //   それ以外 → String
    // 空値が 1 つでもあれば結果を Option<T> で包む
}

fn is_bool_value(s: &str) -> bool {
    matches!(s.to_lowercase().as_str(), "true" | "false")
}

fn is_int_value(s: &str) -> bool {
    s.parse::<i64>().is_ok()
}

fn is_float_value(s: &str) -> bool {
    s.parse::<f64>().is_ok()
}
```

### 2-C: Favnir 型定義フォーマッター

```rust
fn format_inferred_type(ty: &InferredType) -> String {
    match ty {
        InferredType::Int       => "Int".to_string(),
        InferredType::Float     => "Float".to_string(),
        InferredType::Bool      => "Bool".to_string(),
        InferredType::FavString => "String".to_string(),
        InferredType::Option(inner) => format!("Option<{}>", format_inferred_type(inner)),
    }
}

/// InferredTypeDef → Favnir ソースコード文字列
fn format_type_def(def: &InferredTypeDef) -> String {
    // フィールド名の最大長でスペース整列
    // "// auto-generated by `fav infer {source}`\n// Review and adjust before use.\n"
    // "type {Name} = {\n"
    // "    {field}:{padding} {Type}\n"
    // "}\n"
}
```

### 2-D: 型名変換（テーブル名 → PascalCase）

```rust
fn table_name_to_type_name(table: &str) -> String {
    // snake_case → PascalCase
    // 末尾の "s" を除去（単純ヒューリスティック）
    // "users" → "User", "orders" → "Order", "user_profiles" → "UserProfile"
}
```

### 4-B: SQLite テーブル推論

```rust
fn infer_from_sqlite_table(
    conn: &rusqlite::Connection,
    table: &str,
    source_label: &str,
) -> Result<InferredTypeDef, String> {
    // PRAGMA table_info(table) → Vec<(name, type_str, notnull)>
    // sqlite_type_to_inferred → InferredType
    // notnull = 0 → Option<T>
    // type_name = table_name_to_type_name(table)
}

fn sqlite_type_to_inferred(type_str: &str) -> InferredType {
    // "INTEGER"|"INT"|"BIGINT" → Int
    // "REAL"|"FLOAT"|"DOUBLE"|"NUMERIC"|"DECIMAL" → Float
    // "BOOLEAN" → Bool
    // その他 → FavString
}
```

### 4-C: `cmd_infer` SQLite 分岐

```rust
let conn = if conn_str == "sqlite::memory:" { ... } else { ... };
let tables = if let Some(t) = &table_name {
    vec![t.clone()]
} else {
    sqlite_list_tables(&conn)?
};

let defs: Vec<InferredTypeDef> = tables.iter()
    .map(|t| infer_from_sqlite_table(&conn, t, &source_label))
    .collect::<Result<_, _>>()?;

write_infer_multi_output(&defs, out_path.as_deref())?;
```

### 4-D: ファイル出力ヘルパー

```rust
fn write_infer_output(content: &str, out: Option<&str>) {
    // None → print!("{content}")
    // Some(path) → std::fs::write(path, content)
}

fn write_infer_multi_output(defs: &[InferredTypeDef], out: Option<&str>) {
    // None → 全て stdout に空行区切りで出力（1ファイル = 1コメントヘッダー）
    // Some(path) で path が '/' か '\' で終わるか "schema/" 形式 →
    //   ディレクトリとして扱い defs[i] → <dir>/<snake_name>.fav
    // Some(path) でファイルパス → 全定義を1ファイルに連結
}
```

## Phase 6: `--out` ディレクトリサポート + エラー処理整備

**Goal**: `--out schema/` でディレクトリ出力を完成させ、エラーハンドリングを整える。

### 6-A: ディレクトリ出力

```rust
fn write_infer_multi_output(defs: &[InferredTypeDef], out: Option<&str>) {
    match out {
        None => { /* stdout */ }
        Some(p) if p.ends_with('/') || p.ends_with('\\') => {
            std::fs::create_dir_all(p)?;
            for def in defs {
                let filename = format!("{}{}.fav", p, def.name.to_lowercase());
                std::fs::write(filename, format_type_def(def))?;
            }
        }
        Some(p) => {
            let content = defs.iter().map(format_type_def).collect::<Vec<_>>().join("\n");
            std::fs::write(p, content)?;
        }
    }
}
```

## 実装ノート

### `csv` クレートの使用方法

```rust
let mut rdr = csv::Reader::from_path(path)
    .map_err(|e| format!("CSV file not found: {path}: {e}"))?;
let headers: Vec<String> = rdr.headers()?.iter().map(|s| s.to_string()).collect();
let mut columns: Vec<Vec<String>> = vec![vec![]; headers.len()];
for result in rdr.records().take(100) {
    let record = result?;
    for (i, field) in record.iter().enumerate() {
        columns[i].push(field.to_string());
    }
}
```

### `rusqlite` PRAGMA

```rust
let mut stmt = conn.prepare("PRAGMA table_info(?1)")?;
let rows = stmt.query_map([table_name], |row| {
    Ok((
        row.get::<_, String>(1)?,  // name
        row.get::<_, String>(2)?,  // type
        row.get::<_, i32>(3)?,     // notnull
    ))
})?;
```

### フィールド整列アルゴリズム

```rust
let max_name_len = fields.iter().map(|f| f.name.len()).max().unwrap_or(0);
for field in &fields {
    let padding = " ".repeat(max_name_len - field.name.len());
    writeln!(out, "    {}:{} {}", field.name, padding, format_inferred_type(&field.ty))?;
}
```
