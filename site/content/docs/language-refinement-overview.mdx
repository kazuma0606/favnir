# Language Refinement Overview

Favnir v45.1〜v45.9 スプリント「Language Refinement」では、`return` 構文・`match` 完全網羅・型エイリアス・エラーメッセージ改善・数値リテラル・examples 更新を実施し、Favnir の構文を成熟させました。

## 主要機能

### `return` 構文

複数行ボディでの途中脱出専用構文です。単一式の暗黙 return は変更ありません。

```favnir
stage ValidateOrder: Order -> Result<Order> = |order| {
  if order.amount <= 0.0 { return Result.err("invalid amount") }
  Result.ok(order)
}
```

### `match` 完全網羅

- **W034**: 非網羅 match を文（statement）として使う場合の警告
- **E0416**: 非網羅 match を式（value context）として使う場合のハードエラー

```favnir
type Color = Red | Green | Blue

// E0416: non-exhaustive match in value context — Blue is not covered
bind label <- match color {
  Red   => "red"
  Green => "green"
}
```

### 型エイリアス

```favnir
// 透過的エイリアス — UserId と Int は互換
type UserId = Int
fn process(id: UserId) -> Int { id + 1 }

// 不透過型 — OrderId と String は非互換
opaque type OrderId = String
```

### 数値リテラル `_`

可読性向上のため、数値リテラル中に `_` をセパレーターとして使用できます。

```favnir
stage FilterLarge: List<Int> -> List<Int> = |nums| {
  nums |> List.filter(|n| n > 1_000_000)
}

stage CalcRate: Float -> Float = |v| {
  v * 0.000_15
}
```
