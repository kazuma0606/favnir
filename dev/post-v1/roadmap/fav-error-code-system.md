# Favnir Error & Lint System (Post-v1)

日付: 2026-05-01

## 体系化の原則

Favnir のエラーメッセージは「何が間違っているか」だけでなく「どう直すべきか」を提示し、誰が書いても同じ正解に辿り着けるようガイドする。

---

## 1. エラーコード・レンジ

| レンジ | 分類 | 内容 |
|:---|:---|:---|
| `E0100-E0199` | **Syntax** | Lexer / Parser 段階のエラー |
| `E0200-E0299` | **Resolve** | 識別子、インポート、可視性、名前衝突 |
| `E0300-E0399` | **Typing** | 型不一致、推論失敗、ジェネリクス、Capability |
| `E0400-E0499` | **Effect** | エフェクト不足、未定義エフェクト、エフェクト合成エラー |
| `E0500-E0599` | **Context** | 構造上の制約（yield/chain の使用場所、等） |
| `E0600-E0699` | **Schema** | DB スキーマ、invariant、制約矛盾 |
| `E0900-E0999` | **Runtime** | Artifact 形式、VM 実行エラー、WASM 制限 |

---

## 2. 移行マップ（旧コードからのマッピング例）

| 旧コード | 新コード | 理由 |
|:---|:---|:---|
| E001 | E0301 | 最も頻出する型不一致エラーとして中心に配置 |
| E002 | E0201 | 名前解決の基本エラー |
| E004 | E0401 | エフェクト違反の基本エラー |
| E007 | E0411 | Db エフェクト固有エラー（41x シリーズを DB 用に） |
| E032 | E0901 | Artifact 読み込みエラー |

---

## 3. エラーメタデータの構造

セルフホストを見据え、各エラーは以下のメタデータを持つ。

```fav
type ErrorDef = {
    code:        String
    title:       String
    description: String
    bad_example: String
    fix_example: String
}
```

これにより、`fav explain` はもちろん、LSP を通じてエディタ上で直接修正案を表示することが可能になる。

---

## 4. Lint システム (`Lxxxx`)

「動くが、Favnir の規約 (CoC) に反する」コードを指摘する。

- **`L1000`**: パイプラインの複雑度過大（1つの `seq` が長すぎる）。
- **`L2000`**: DB 連携型において `primary_key` が定義されていない。
- **`L3000`**: 関数名・変換名が推奨される命名規則（PascalCase/snake_case）に従っていない。

---

## 5. セルフホストへの統合

セルフホストされたコンパイラにおいて、エラー定義自体を `.fav` ファイルとして管理する。
コンパイラをビルドする際、これらのエラー定義からドキュメントとエラー定数を自動生成する。

---

## 一言でいうと

> エラーコードは単なる番号ではなく、ユーザーを正しい設計（CoC）へと導く「地図」である。
