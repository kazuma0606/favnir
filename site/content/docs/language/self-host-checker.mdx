# Self-Host Checker

`fav/self/checker.fav` — Favnir で実装された型チェッカーです。

v7.7.0 では **エフェクト追跡** / **全 builtin シグネチャ登録** / **エラーコード（E0xxx）** /
**match 網羅性チェック（Option / Result）** を実装し、Rust 版 `checker.rs` の基本機能と同等になりました。

## アーキテクチャ

```
fav check foo.fav
  └─ main.rs → Checker::check_program()    [Rust — 現在の実装]

fav run self/checker.fav -- ...            [将来: checker.fav が代替]
  └─ check(prog: Program) -> Result<String, String>
       ├─ infer_expr(expr, env)            型推論パス
       ├─ infer_expr_effects(expr)         エフェクト収集パス
       └─ check_match_exhaustive(ty, arms) 網羅性チェックパス
```

## 3 つの解析パス

### 1. 型推論パス (`infer_expr`)

式を再帰的に走査して型名（`"Int"` / `"String"` / `"Option"` 等）を返します。

```favnir
fn infer_expr(expr: Expr, env: List<KVPair>) -> Result<String, String>
```

`KVPair { first: name, second: type_str }` の association list を環境として持ち回ります。

### 2. エフェクト収集パス (`infer_expr_effects`)

式から使用エフェクトを収集します。Result を返さない純粋関数です。

```favnir
fn infer_expr_effects(expr: Expr) -> List<String>
```

| ECall の namespace | 生成エフェクト |
|-------------------|---------------|
| `"IO"` / `"Compiler"` | `"IO"` |
| `"Cache"` | `"Cache"` |
| `"Queue"` | `"Queue"` |
| `"Email"` | `"Email"` |
| その他 | なし |

### 3. 網羅性チェックパス (`check_match_exhaustive`)

`EMatch` の scrutinee 型と arms を受け取り、非網羅なら E0004 エラーを返します。

```favnir
fn check_match_exhaustive(scrut_ty: String, arms: Expr) -> Option<String>
```

| scrutinee 型 | 必要なコンストラクタ |
|-------------|---------------------|
| `"Option"` | `None` + `Some`（または `_`） |
| `"Result"` | `Ok` + `Err`（または `_`） |
| その他 | チェックなし（v7.7.0 スコープ外） |

## エラーコード一覧

| コード | 意味 | 例 |
|--------|------|----|
| `E0001` | arithmetic 型ミスマッチ | `Int + String` |
| `E0002` | logical 演算子に Bool 以外 | `1 && "a"` |
| `E0003` | undeclared effect | `!IO` なしで `IO.println` を呼ぶ |
| `E0004` | non-exhaustive match | Option に `Some` アームがない |

エラーは `fmt_err("E0001", "...")` で `"E0001: ..."` 形式にフォーマットされます。

## FnDef の effects フィールド

v7.7.0 で `FnDef` に `effects: List<String>` フィールドが追加されました。

```favnir
type FnDef = {
    is_public: Bool
    name: String
    params: List<Param>
    ret: TypeExpr
    effects: List<String>   // ["IO", "Cache"] 等
    body: Expr
}
```

`check_fn_def` は `infer_expr_effects(body)` で推論したエフェクトを
`fd.effects` と照合し、宣言されていないエフェクトがあれば `E0003` を返します。

## v7.8.0: ジェネリクス対応

### 型変数と置換環境

`TyVar(String)` を `Type` sum type に追加。置換環境は `List<KVPair>` で表現。

```favnir
// 型変数 "A" を "Int" に置換
bind subst <- subst_insert(subst_empty(), "A", "Int")
apply_subst(subst, "A")  // → "Int"
```

### 基本単一化 (`unify`)

```favnir
unify("A", "Int", subst_empty())    // Ok([{A → Int}])
unify("Int", "Int", subst_empty())  // Ok([])
unify("Int", "String", subst_empty()) // Err("E0005: cannot unify Int with String")
```

| ケース | 結果 |
|--------|------|
| `t1 == t2` | `Ok(subst)`（変更なし） |
| `t1` or `t2` == `"Unknown"` | `Ok(subst)`（後方互換） |
| `is_type_var(t1)` | subst に `t1 → t2` を追加 |
| `is_type_var(t2)` | subst に `t2 → t1` を追加 |
| 競合（既存バインドと不一致） | `Err("E0005: conflict: ...")` |
| それ以外 | `Err("E0005: cannot unify ...")` |

### Parameterized 型文字列

```favnir
type_str_outer("List<Int>")   // → "List"
type_str_inner("List<Int>")   // → "Int"
type_str_inner("Option<String>") // → "String"
```

### ジェネリクス対応 builtin 推論

`List.*` / `Option.*` / `Result.*` 呼び出しで引数型から戻り型を精密化:

```favnir
// List.first(xs) で xs: List<Int> → Option<Int>
infer_generic_list("first", ["List<Int>"])  // → "Option<Int>"
infer_generic_list("filter", ["List<String>", "Fn"])  // → "List<String>"
```

| 関数 | 推論ロジック |
|------|-------------|
| `List.first(List<T>)` | `Option<T>` |
| `List.find(List<T>, Fn)` | `Option<T>` |
| `List.filter(List<T>, Fn)` | `List<T>` |
| `List.push(List<T>, T)` | `List<T>` |
| `List.singleton(T)` | `List<T>` |
| `List.map(List<T>, Fn)` | `"List"`（HM は v7.9.0） |

### ユーザー定義ジェネリクス型

`TypeDef` に `type_params: List<String>` フィールドを追加。
`type_param_env(params, args)` で型パラメータ → 型引数のマッピングを構築。

```favnir
type_param_env(["A", "B"], ["Int", "String"])
// → [{A → Int}, {B → String}]
```

## v7.9.0: HM 型推論（完全版）

### occurs_in — 無限型防止

型変数が型文字列中に出現するかを確認します。`unify_deep` の occurs check に使用。

```favnir
occurs_in("A", "A")           // true
occurs_in("A", "Int")         // false
occurs_in("A", "List<A>")     // true
occurs_in("A", "List<Int>")   // false
```

### unify_deep — ネスト型の単一化

v7.8.0 の `unify`（flat 型のみ）を拡張し、`"List<A>"` vs `"List<Int>"` のようなパラメータ化型を再帰的に単一化します。

```favnir
unify_deep("List<A>", "List<Int>", subst_empty())   // Ok([A→Int])
unify_deep("List<Int>", "List<Int>", subst_empty()) // Ok([])
unify_deep("List<A>", "Option<Int>", subst_empty()) // Err("E0005: cannot unify List with Option")
unify_deep("A", "List<A>", subst_empty())           // Err("E0006: occurs check failed: A in List<A>")
```

| ケース | 結果 |
|--------|------|
| `t1 == t2` | `Ok(subst)` |
| `is_type_var(t1)` + occurs check OK | subst に追加 |
| `is_type_var(t1)` + occurs check NG | `Err("E0006: occurs check failed: ...")` |
| 両方がパラメータ化型 + outer 一致 | inner を再帰的に `unify_deep` |
| 両方がパラメータ化型 + outer 不一致 | `Err("E0005: cannot unify outer1 with outer2")` |

### InfState / InfResult — 推論状態レコード

```favnir
type InfState = {
    subst:   List<KVPair>   // 型変数置換環境
    counter: Int             // fresh 変数カウンタ
}

type InfResult = {
    ty:      String          // 推論結果型
    subst:   List<KVPair>
    counter: Int
}
```

### fresh_var — 新鮮型変数生成

```favnir
fresh_var(0)   // "t0"
fresh_var(10)  // "t10"
```

### infer_hm — HM 型推論パス

状態を引き回しながら式を型推論します。

```favnir
fn infer_hm(expr: Expr, env: List<KVPair>, state: InfState) -> Result<InfResult, String>
```

| ノード | 処理 |
|--------|------|
| `ELit(_)` | リテラル型を直接返す |
| `EVar(name)` | env にあれば型を返す、なければ `fresh_var` を生成（counter+1） |
| `EBind(name, val, body)` | val を推論 → env に追加 → body を推論（状態スレッディング） |
| `EIf(cond, then, else)` | then/else を推論 → `unify_deep` で型を統合 |
| それ以外 | `infer_expr` にフォールバック |
| `ECall(ns, fname, args)` | `infer_call_hm` で処理（v8.0.0 以降） |

`check_fn_def` は `infer_expr` から `infer_hm` に切り替え済み。

## v8.0.0: Let ポリモーフィズム（型スキーム generalization / instantiation）

v8.0.0 では `checker.fav` に **Let 多相（Parametric Polymorphism）** を実装した。
ユーザー定義ジェネリクス関数の呼び出しを `infer_hm` が正しく推論できる。

### 型スキーム文字列形式

環境 `List<KVPair>` は `String → String` の連想リストのため、TypeScheme を以下の形式で直列化する。

```
"forall|<vars>|<params>|<ret>"
```

| セクション | 内容 |
|------------|------|
| `"forall"` | スキームであることの識別子 |
| `<vars>` | `,` 結合の型変数名（例: `"A,B"`） |
| `<params>` | `;` 結合のパラメータ型（例: `"List<A>;Fn"`） |
| `<ret>` | 戻り型文字列（例: `"Option<A>"`） |

型変数を持たない関数（monotype）はそのまま戻り型文字列を格納する。

### 新規関数

| 関数 | 説明 |
|------|------|
| `is_type_var_extended(s)` | 1文字 A〜Z または 2文字（`T0`〜`Z9`）の型変数か判定 |
| `collect_type_vars_from_te(te)` | TypeExpr から型変数名を再帰収集 |
| `list_dedup(lst)` | リストの重複除去（初出順を維持） |
| `fn_to_scheme_str(fd)` | FnDef からスキーム文字列を構築 |
| `collect_fn_schemes(items, env)` | 全 IFn をスキャンしてスキーム env を構築（pre-pass） |
| `build_scheme_subst_inner(param_tys, arg_tys, subst)` | `unify_deep` をリストで繰り返し適用 |
| `apply_scheme_subst(ret_ty, vars, subst)` | `String.replace` で型変数を具体化 |
| `instantiate_fn_scheme(scheme, arg_tys, state)` | スキームを実引数型で具体化 |
| `infer_call_user(fname, args, env, state)` | ユーザー定義関数の呼び出し推論 |
| `infer_call_hm(ns, fname, args, env, state)` | `ns=""` なら `infer_call_user`、それ以外は既存 `infer_call` にフォールバック |

### 動作例

```favnir
fn first_elem(xs: List<A>) -> Option<A> {
    List.first(xs)
}

// infer_hm が first_elem([1,2,3]) の戻り型を "Option<Int>" と推論できる
```

`check` 関数は `collect_fn_schemes` で事前に全関数スキームを収集してから `check_items` を呼ぶ。
