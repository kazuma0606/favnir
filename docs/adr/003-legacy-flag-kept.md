# ADR-003: `--legacy` フラグは削除せず非推奨に留める

- **状態**: 採用
- **決定日**: v8.5.0（`--legacy` 導入時）
- **更新**: v9.0.0（非推奨化・削除しないことを明示）

---

## 背景

v8.5.0 で `fav run` のデフォルトが Favnir パイプライン（`compiler.fav`）に切り替わった。
それ以前の Rust パイプライン（`fav/src/` の Rust 実装）へのアクセス手段として
`--legacy` フラグが追加された。

v9.0.0 でセルフホスト完成宣言を行った際、`--legacy` の扱いを決定する必要があった。

## 決定

`--legacy` フラグは削除せず、非推奨（deprecated）として維持する。

## 理由

1. **デバッグ手段の確保**: Favnir パイプラインにバグが生じた場合、
   `--legacy` で Rust パイプラインと挙動を比較できる。
   コンパイラのバグ修正に不可欠なフォールバック手段。

2. **段階的移行の安全弁**: セルフホストが完全に安定するまでの間、
   既存ユーザーへの影響を最小化する。

3. **削除コストが低い**: フラグを残してもコードベースの複雑さはほぼ増えない。
   削除した場合のリスク（デバッグ手段の喪失）と天秤にかけて残す方が合理的。

## 却下した選択肢

- **v9.0.0 で削除する**: デバッグ手段を失うリスクが高い。Bootstrap 安定後に再検討可能。
- **`--rust` にリネーム**: 互換性が壊れる。`--legacy` の意味は明確なので変更不要。

## 現在の動作

```bash
fav run hello.fav           # Favnir パイプライン（デフォルト）
fav run hello.fav --legacy  # Rust パイプライン（非推奨、警告あり）
fav check hello.fav         # checker.fav 経由
fav check hello.fav --legacy  # Rust checker 経由（非推奨）
```

## 影響

- `--legacy` は `INVARIANTS.md` II-2 で削除禁止として記載
- Rust パイプラインのコードは `fav/src/` に維持される
- 将来的な削除には、Bootstrap が長期間安定していることと
  デバッグ手段の代替（例: `fav debug --diff-pipelines`）の整備を条件とする
