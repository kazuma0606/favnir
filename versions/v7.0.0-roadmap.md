作成日: 2026-05-19

---

## 設計哲学（v6 からの継続）

```
Rust VM = 接続・生データのやり取りのみ（永続 primitive）
Favnir  = 意味のある操作はすべて Favnir で実装
```

VM primitive は薄く保ち、その上に Favnir 製の豊かなエコシステムを積み上げる。
Lua/C や Erlang/BEAM と同じ分割。

---

## エフェクト設計方針（v7 で確定）

DB は `!AWS` と異なり、**読み取り・書き込み・スキーマ変更の意味が明確に違う**
ため、エフェクトを 3 段階に分ける:

```favnir
fn get_user(id: Int) -> Option<User> !DbRead      // SELECT のみ → 副作用なし
fn archive_user(id: Int) -> Unit    !DbWrite      // INSERT/UPDATE/DELETE
fn drop_table(name: String) -> Unit !DbAdmin      // DDL (CREATE/DROP/ALTER)
```

`fav explain` で自動データリネージが得られる:

```
$ fav explain pipeline.fav
Effects: !DbRead(users, orders), !DbWrite(audit_log), !Io
→ 「users と orders を読んで audit_log に書く」が静的に保証される
```

---

## 層の構造

```
ユーザーコード
  bind users <- Db.find_all<User>(where: |u| u.active == true)  !DbRead
  bind _     <- Db.insert(audit_log)                             !DbWrite

runes/db/ (Favnir ORM) ← runes/db/ に既存コードあり
  find_all<T> / find_by_id<T> / insert<T> / update<T> / delete<T>
  QueryBuilder / Migration / Transaction
  スキーマ ↔ Favnir 型の対応管理

VM primitive (Rust・永続)
  Db.connect_raw(url) -> Result<DbHandle, String>
  Db.query_raw(handle, sql, params) -> Result<List<Map>, String>
  Db.exec_raw(handle, sql, params) -> Result<Int, String>
```

## v7.1.0 — `fav explain` 強化（データリネージ）

### 現状

`fav explain` はエラー説明コマンドとして実装済み。

### v7.1.0 拡張

```bash
fav explain --lineage pipeline.fav
```

出力例:
```
Sources:
  !DbRead   → users (PostgreSQL), orders (PostgreSQL)
  !Io       → /data/config.csv

Sinks:
  !DbWrite  → audit_log (PostgreSQL)
  !AWS(S3)  → s3://my-bucket/reports/

Transformations:
  users ──→ filter(active) ──→ join(orders) ──→ audit_log
```

エフェクトシグネチャを静的に解析して生成。
数百万円のデータカタログと同等の情報をコードから自動生成（製品戦略の核心）。

## v7.2.0 — スキーマ推論（DB → Favnir 型自動生成）

```bash
fav schema infer --db postgresql://localhost/mydb --table users
```

出力例:
```favnir
// 自動生成: users テーブルから
type User = {
  id:         Int
  name:       String
  email:      String
  created_at: String
  active:     Bool
}
```

- `Db.describe_table_raw(handle, table)` VM primitive を追加
- スキーマ → Favnir 型変換ロジックを Favnir で実装
- `fav schema infer` コマンドを追加

これにより「既存 DB を今日モダン API にする」レガシー統合ピッチが完成。

## v7.3.0 — SQL Rune（型安全クエリビルダ）

dbt の代替。型安全なクエリを Favnir で記述できる。

```favnir
import rune "sql"

// 型安全なクエリビルダ
bind result <- Sql.from<User>()
  |> Sql.where(|u| u.active == true)
  |> Sql.join<Order>(|u, o| u.id == o.user_id)
  |> Sql.select(|u, o| { user: u.name, total: o.amount })
  |> Sql.limit(100)
  |> Sql.run;                                    // !DbRead
```

- クエリビルダを Favnir で実装（`Db.query_raw` の上に構築）
- 型パラメータでスキーマ整合性をコンパイル時に検証
- `!DbRead` / `!DbWrite` エフェクト自動付与

## v7.4.0 — Rune エコシステム拡充

同じ「VM primitive = 薄い接続層、Favnir = 意味のある操作」パターンで拡張:

| Rune | VM primitive（Rust） | Favnir 層 |
|------|---------------------|----------|
| `queue` | `Queue.send_raw` / `Queue.recv_raw` | バッチ送信、dead letter、ack 管理 |
| `cache` | `Cache.get_raw` / `Cache.set_raw` | TTL 管理、invalidation パターン |
| `fs` | `IO.read_file_raw`（v5.1.0 追加済み）| ディレクトリ walk、glob、watch |
| `slack` | `Http.send_raw`（既存）| Slack 通知 Rune |
| `email` | `Smtp.send_raw` | メール送信 |
