# Import Syntax 2.0

v48.1.0 で導入された新しい import 構文（import 2.0）を説明します。
旧構文（`import rune "X"`）は v48.5.0 から W035 警告対象となっています。

## パッケージ import

`fav.toml` の `[runes]` に宣言したパッケージを直接 import できます。

パッケージ名をそのまま指定します（例: `kafka`、`postgres`）。

## ローカル import

相対パスで同プロジェクト内の `.fav` ファイルを参照できます（例: `"./stages/validate"` を `as validate` で使用）。

## fav.toml との連携

import するパッケージは `fav.toml` の `[runes]` セクションに宣言が必要です。
宣言のない import は E0418 エラーになります。

```toml
[runes]
kafka   = "2.1.0"
postgres = "1.0.0"
```

## 旧構文からの移行（W035）

旧構文を使い続けると W035 警告が出ます。

旧構文（`import rune "kafka"` 形式）は **W035** 警告が出ます。
新構文（`import kafka` 形式）への移行を推奨します。

詳細な移行手順は [Import Migration Guide](/docs/migration-guide-import) を参照してください。
