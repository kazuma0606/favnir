# Favnir Data Lineage & Explainable Graphs

日付: 2026-05-02

## 概要

Favnir は「なぜこの結果になったのか」を証明する責任を言語レベルで担う。
SSS アーキテクチャ（State-Stage-Sequence）の規約を利用し、データの加工履歴（Lineage）を自動追跡・可視化する仕組みを提供する。

---

## 1. 計算履歴の自動追跡 (Lineage Tracking)

`Sequence` 内を流れるデータは、背後で「どの Stage を通り、どのような演算を施されたか」という計算グラフの履歴を保持する。

*   **`Tracked<T>`**: 値 `T` と、その生成までの計算ステップをカプセル化した内部状態。
*   **メタデータの伝播**: 開発者が意識することなく、パイプラインの結合（`|>`）に従って履歴が自動的に蓄積される。

---

## 2. `fav explain result`

特定の計算結果に対して、その「家系図」を表示する。

```bash
# 結果 121.0 の由来を遡る
$ fav explain result --value 121.0

[Result: 121.0]
  <- Stage: TaxCalculation (input * 1.1)
    <- Stage: PriceSum (fold `+`)
      <- Data: sales.csv (Column: price, Row: 42)
```

### メリット
- **監査性**: 解析結果の根拠を非エンジニアにも説明できる。
- **デバッグ**: 異常値が「どの入力データ」または「どの計算ステップ」で発生したかを即座に特定できる。

---

## 3. 計算グラフの可視化 (Graph Visualization)

`Sequence` の構成をグラフとして書き出し、データのフローと数学的性質を俯瞰する。

*   **`fav graph`**: プロジェクト全体のデータフローを可視化。
*   **ボトルネック・エラー伝播の特定**: どの Stage で情報量が失われているか、またはエラー率が高いかをグラフ上にヒートマップとして表示する。

---

## 4. 信頼の連鎖 (Chain of Trust)

自動追跡機能は、以前議論した「データソースとの統合」と組み合わさることで、真価を発揮する。

- **起源 (Origin)**: `db.fetch` や `csv.parse` が、データの物理的な起源（URI, コミットハッシュ等）を Lineage の起点として記録する。
- **正当性の証明**: 最終的な出力レポートに「この数値は、この日時のこのデータから、このアルゴリズムで導き出されたものである」という電子署名付きの証明書を付与することも可能になる。
