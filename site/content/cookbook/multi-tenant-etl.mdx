# マルチテナント ETL

`runes/tenant` が提供する Tenant Rune を使って、テナントごとに DB スキーマと S3 ストレージを分離した ETL パイプラインを構築します。

## コード例

```favnir
import runes/tenant

fn run_tenant_etl(ctx: AppCtx, allowed_tenants: List<String>) -> Result<Unit, String> {
    bind _      <- tenant.validate_tenant(ctx, allowed_tenants)
    bind schema <- tenant.db_schema(ctx)
    bind prefix <- tenant.s3_prefix(ctx)
    bind rows   <- extract_from_db(schema)
    bind _      <- upload_to_s3(prefix, rows)
    IO.println("ETL complete")
    Result.ok(unit)
}
```

## ポイント

- `tenant.validate_tenant` でテナント ID が許可リストに含まれるか検証します。不正なテナントは `Result.err` で早期リターンします。
- `tenant.db_schema` はテナントごとの DB スキーマ名（例: `tenant_acme`）を返します。クエリのスキーマプレフィックスとして使用します。
- `tenant.s3_prefix` はテナントごとの S3 パスプレフィックス（例: `tenants/acme/`）を返します。ファイルのアップロード先パスに使用します。
- `AppCtx` にテナント ID を持たせることで、全パイプライン関数が同一のテナントコンテキストで動作します。
