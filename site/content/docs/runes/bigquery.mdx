# BigQuery Rune

`import rune "bigquery"` で Google BigQuery への接続・クエリ・挿入・テーブル管理ができます。

> **注**: v27.4.0 は引数検証 stub 実装です。`google-cloud-bigquery` クレートを使った実接続は v28.x で実装予定です。

## セットアップ

```toml
# fav.toml
[runes]
bigquery = "27.4.0"
```

```favnir
import rune "bigquery"
```

## エフェクト

BigQuery は DB 接続プロトコル経由の操作のため `!Db` エフェクトを使用します（ClickHouse / Redshift と同一）。v15.2.0 の `!Gcp` エフェクトから変更されています。

## 関数リファレンス

### `BigQuery.connect`

```favnir
fn connect(config: String) -> Result<String, String> !Db
```

BigQuery への接続を確立し、接続ハンドル文字列を返します。`config` にはプロジェクト ID・データセット等の接続情報を渡します。

```favnir
bind conn <- BigQuery.connect("project:my-project,dataset:analytics")
```

### `BigQuery.query`

```favnir
fn query(conn: String, sql: String) -> Result<String, String> !Db
```

SELECT クエリを実行し、結果を JSON 配列文字列として返します。

```favnir
bind rows <- BigQuery.query(conn, "SELECT count(*) FROM analytics.events")
```

### `BigQuery.insert`

```favnir
fn insert(conn: String, table: String, rows: String) -> Result<Unit, String> !Db
```

streaming insert でデータをテーブルに挿入します（即時反映、小量向き）。

```favnir
bind _ <- BigQuery.insert(conn, "events", rows_json)
```

### `BigQuery.load_from_gcs`

```favnir
fn load_from_gcs(conn: String, table: String, gcs_uri: String, format: String) -> Result<Unit, String> !Db
```

GCS からバルクロードします（大量データ向き）。`format` は `"PARQUET"` / `"CSV"` / `"AVRO"` 等を指定します。

```favnir
bind _ <- BigQuery.load_from_gcs(conn, "events", "gs://my-bucket/events/*.parquet", "PARQUET")
```

### `BigQuery.create_table`

```favnir
fn create_table(conn: String, table: String, schema: String) -> Result<Unit, String> !Db
```

スキーマ定義 JSON を使ってテーブルを作成します。

```favnir
bind _ <- BigQuery.create_table(conn, "events", "{\"fields\":[{\"name\":\"id\",\"type\":\"INTEGER\"}]}")
```

## パイプライン例

```favnir
// examples/bigquery_analytics.fav
import rune "bigquery"

stage CreateEventTable: Unit -> Result<Unit, String> !Db = |_| {
    bind conn <- BigQuery.connect("project:my-project,dataset:analytics")
    BigQuery.create_table(conn, "events", "{\"fields\":[{\"name\":\"id\",\"type\":\"INTEGER\"}]}")
}

// seq pipeline は前ステージの成功値（Unit）を次ステージの引数として渡す
stage LoadFromGcs: Unit -> Result<Unit, String> !Db = |_| {
    bind conn <- BigQuery.connect("project:my-project,dataset:analytics")
    BigQuery.load_from_gcs(conn, "events", "gs://my-bucket/events/*.parquet", "PARQUET")
}

stage QueryStats: Unit -> Result<String, String> !Db = |_| {
    bind conn <- BigQuery.connect("project:my-project,dataset:analytics")
    BigQuery.query(conn, "SELECT count(*) FROM analytics.events")
}

seq BigQueryAnalyticsPipeline = CreateEventTable |> LoadFromGcs |> QueryStats
```

## v15.2.0 からの移行

| 変更点 | v15.2.0（旧） | v27.4.0（新） |
|---|---|---|
| エフェクト | `!Gcp` | `!Db` |
| API スタイル | `fn query(project_id, dataset, sql)` | `fn connect(config)` + `fn query(conn, sql)` |
| 関数の可視性 | `fn`（非 public） | `public fn` |

## スコープ外（v28.x 以降）

- `google-cloud-bigquery` クレートを使った実 BigQuery 接続
- `BigQuery.query[T]` ジェネリック API（型安全な行取得）
- service account JSON の実認証
- BigQuery Emulator（`ghcr.io/goccy/bigquery-emulator`）との実通信
- `load_from_gcs` の format バリデーション（CSV / PARQUET / AVRO / ORC 等）
- streaming insert の quota 制限対応
