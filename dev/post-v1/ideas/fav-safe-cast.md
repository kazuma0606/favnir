# Favnir Safe Type Conversion (Casting) Design

日付: 2026-05-01

## 概要

Favnir における型変換（キャスト）は、単なる低レベルなビット操作ではなく、データをある状態（State）から別の状態へ進める **「工程（Stage）」** として扱う。

既存言語の `(int)v` や `as` 形式のキャストは、精度損失やランタイムエラーを隠蔽し、データ解析の信頼性を損なう可能性があるため、Favnir では以下の原則を採用する。

---

## 1. 失敗を許容する変換 (Fallible Stage)

文字列から数値への変換や、範囲の狭い型への変換など、失敗する可能性がある変換は常に **Result 型 (`!`) を返す Stage** として定義する。

```fav
-- String から Int への変換
stage ToInt: String -> Int! = |s| { ... }

-- 使用例: chain による安全な伝播
chain n <- "123" |> String.to_int
```

### 規約 (CoC)
- 失敗の可能性がある変換には必ず `!` を付けることを強制する。
- これにより、パイプラインの途中でデータが不正になることを防ぎ、エラーハンドリングを型レベルで強制する。

---

## 2. 型コンストラクタによる記法 (Sugar)

`Int(s)` や `Double(s)` といった記法を、標準的な `parse` ステージ呼び出しのシンタックスシュガーとして提供する。

```fav
bind n <- Int("123") -- 内部的には String.to_int("123") の呼び出し
```

これは「新しい状態（State）を構築する」というメンタルモデルに合致する。

---

## 3. 精度損失への明示的アプローチ

精度が失われる変換（例：`Double` -> `Int`）において、暗黙的な切り捨ては行わない。開発者は意図を明確にするために、具体的な丸め戦略を持つ Stage を選択しなければならない。

- `Double.floor`: 小数点以下切り捨て
- `Double.round`: 四捨五入
- `Double.ceil`: 切り上げ

```fav
bind n <- 11.7 |> Double.floor -- 11
```

---

## 4. セマンティック・プロモーション (Invariant 統合)

プリミティブ型から `invariant` を持つ高度な State（型）への変換は、自動的にバリデーションを伴う。

```fav
type Age {
    value: Int
    invariant value >= 0
}

-- キャスト時に不変条件がチェックされる
chain age <- -5 |> Age.from_int -- invariant 違反により Err となる
```

---

## 5. キャストの分類まとめ

| 種類 | 例 | 戻り値 | 規約 |
|:---|:---|:---|:---|
| **昇格 (Widening)** | `Int` -> `Double` | `T` (安全) | 精度が失われないため暗黙的/直接変換を許可 |
| **降格 (Narrowing)** | `Double` -> `Int` | `T!` (要注意) | 丸め戦略の明示を要求する |
| **解析 (Parsing)** | `String` -> `Int` | `T!` (失敗可) | 常に Result 型を返し `chain` を強制する |
| **契約 (Contract)** | `Int` -> `Age` | `T!` (制約) | `invariant` チェックを自動実行する |

---

## 一言でいうと

> Favnir に「魔法のキャスト」は存在しない。
> 全ての型変換は、成功か失敗かが型で示された「明示的な工程（Stage）」である。
