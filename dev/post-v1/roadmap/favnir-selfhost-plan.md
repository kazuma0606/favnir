# Favnir Self-Hosting Plan: Incremental Independence

日付: 2026-05-02

## 概要: ハイブリッド・セルフホスト

Favnir は、Rust の「筋肉（実行基盤・メモリ安全性）」と Favnir の「知能（コンパイラロジック・説明可能性）」を組み合わせた **ハイブリッド・セルフホスト** を目指す。

コンパイラの複雑なロジックを Favnir 自身で記述することで、「説明可能なコンパイラ」を実現しつつ、実行時の底力と安全性は Rust 製の VM（Virtual Machine）に委ねる。

---

## 1. 役割分担：知能 vs 筋肉

| 役割 | 担当言語 | 理由 |
|:---|:---|:---|
| **知能 (Compiler)** | **Favnir** | SSS 構造による見通しの良さ、ドッグフーディングによる言語改善、ビルド高速化。 |
| **筋肉 (VM/Runtime)** | **Rust** | 借用チェッカーによる確実なメモリ安全性、低レイヤの最適化、OS/ハードウェアとの接続。 |

この構成により、Rust の重厚なビルド時間を待つことなく、Favnir の修正を Favnir 上で即座にテストできる高速な開発サイクルを実現する。

---

## 2. 段階的自立 (Incremental Independence) のステップ

### Step 1: パーサーの移植 (Frontend)
- **対象**: 字句解析、構文解析。
- **目的**: Favnir のソースを Favnir のデータ構造（ADT）に変換する処理をセルフホスト化。
- **実行**: Rust 製の VM 上で Favnir 製パーサーを動かす。

### Step 2: 型チェッカーの移植 (Middleend)
- **対象**: 型推論、Invariant チェック、Effect 解析。
- **目的**: Favnir の「正しさ」を定義するコアロジックを Favnir で記述。
- **利点**: コンパイラ自体が `invariant` によって保護され、「世界で最もバグの少ないコンパイラ」の土台ができる。

### Step 3: コード生成の移植 (Backend)
- **対象**: バイトコード生成、WASM 生成。
- **目的**: Rust を介さず、Favnir から直接実行バイナリを出力。
- **結果**: `fav build` コマンドが内部で Favnir 製コンパイラを（Rust VM 上で）呼び出す状態。

---

## 3. セルフホストがもたらす本質的価値

1.  **説明可能なコンパイル工程**: 
    - `fav explain compiler` により、ソースコードがどの `Stage` を通って変換されたかを可視化できる。
2.  **安全性と速度の両立**: 
    - Rust の借用チェッカーが VM を守り、VM が Favnir の実行を守る。コンパイラ開発者は Rust の複雑さに悩まされることなく、ロジックに集中できる。
3.  **ポータビリティの向上**: 
    - コンパイラ自体が Favnir（または WASM）で動くため、ブラウザやエッジ環境でのコンパイル・実行が容易になる。

---

## 4. リスクと対策

| リスク | 対策 |
|:---|:---|
| **Rust VM の肥大化** | VM は最小限のプリミティブに絞り、複雑な機能は `rune` (Favnir製) として実装する。 |
| **性能の低下** | 計算密度の高い処理は Rust 側のビルトイン関数に委譲し、ロジックの制御のみを Favnir が担う。 |
| **鶏と卵の問題** | v1.0.0 の Rust 製コンパイラを「ブートストラップ用」として永久保存し、いつでも一からビルドできるようにする。 |

---

## 結論

Favnir のセルフホストは、Rust を捨てることではない。
**「Rust を最強のインフラとして使い倒しつつ、言語の魂（ロジック）を Favnir 自身で定義する」** という、実利と理想を両立した進化の道である。
