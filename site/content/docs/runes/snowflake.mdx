# Snowflake Rune

エフェクト: `!Snowflake`

```favnir
import rune "snowflake"
```

Snowflake SQL API v2（REST）経由でクエリ・DML を実行できる Rune。
JWT RS256 認証（RSA キーペア）を使用する。追加ライブラリは不要。

---

## fav.toml 設定

プロジェクトルートの `fav.toml` に `[snowflake]` セクションを追加する。
値には `${ENV_VAR}` 形式の環境変数参照を書ける。

```toml
[snowflake]
account   = "${SNOWFLAKE_ACCOUNT}"   # 例: xy12345.ap-northeast-1.aws
user      = "${SNOWFLAKE_USER}"
warehouse = "COMPUTE_WH"
role      = "SYSADMIN"
database  = "MY_DB"
schema    = "PUBLIC"
```

---

## 環境変数

| 変数名 | 説明 | 必須 |
|---|---|---|
| `SNOWFLAKE_ACCOUNT` | アカウント識別子（例: `xy12345.ap-northeast-1.aws`） | ✓ |
| `SNOWFLAKE_USER` | ユーザー名 | ✓ |
| `SNOWFLAKE_PRIVATE_KEY` | RSA 秘密鍵（PEM 形式） | ✓ |
| `SNOWFLAKE_ROLE` | 実行ロール（省略時: デフォルトロール） | |
| `SNOWFLAKE_WAREHOUSE` | ウェアハウス名 | |
| `SNOWFLAKE_DATABASE` | データベース名 | |
| `SNOWFLAKE_SCHEMA` | スキーマ名 | |

RSA キーペアの生成:

```bash
openssl genrsa -out private_key.pem 2048
openssl rsa -in private_key.pem -pubout -out public_key.pem
```

Snowflake ユーザーへの公開鍵登録:

```sql
ALTER USER <USER> SET RSA_PUBLIC_KEY='<公開鍵（ヘッダー/フッター除く）>';
```

## API リファレンス

### execute

```favnir
execute(sql: String) -> Int !Snowflake
```

DML（INSERT / UPDATE / DELETE / CREATE 等）を実行し、影響行数を返す。

```favnir
stage CreateTable: Unit -> Int !Snowflake = |_| {
  snowflake.execute("CREATE TABLE IF NOT EXISTS orders (id INT, customer STRING, amount FLOAT)")
}

stage InsertOrders: List<Order> -> Int !Snowflake = |orders| {
  bind json_str <- Json.encode_raw(orders)
  snowflake.execute(
    $"INSERT INTO orders SELECT value:id::INT, value:customer::STRING, value:amount::FLOAT FROM TABLE(FLATTEN(PARSE_JSON('{json_str}')))"
  )
}
```

### query

```favnir
query<T>(sql: String) -> List<T> !Snowflake
```

SELECT クエリを実行し、結果を `List<T>` として返す。
戻り値の型 `T` は Favnir のレコード型で宣言する。

```favnir
type Order   = { id: Int  customer: String  amount: Float }
type Summary = { region: String  total: Float  count: Int }

stage FetchOrders: Unit -> List<Order> !Snowflake = |_| {
  snowflake.query<Order>("SELECT id, customer, amount FROM orders ORDER BY id")
}

stage SumByRegion: Unit -> List<Summary> !Snowflake = |_| {
  snowflake.query<Summary>(
    "SELECT region, SUM(amount) AS total, COUNT(*) AS count FROM orders GROUP BY region"
  )
}
```

## fav infer — スキーマ自動生成

Snowflake テーブルの `INFORMATION_SCHEMA.COLUMNS` を参照して
Favnir 型定義を自動生成できる。

```bash
fav infer --from snowflake --table ORDERS
```

出力例:

```favnir
// Generated by fav infer --from snowflake --table ORDERS
type Orders = { order_id: Int  customer: String  amount: Float  region: String }
```

`--out` オプションでファイルに保存:

```bash
fav infer --from snowflake --table ORDERS --out src/types/orders.fav
```

Snowflake 型 → Favnir 型のマッピング:

| Snowflake 型 | Favnir 型 |
|---|---|
| NUMBER / INTEGER / BIGINT | `Int` |
| FLOAT / DOUBLE | `Float` |
| BOOLEAN | `Bool` |
| VARCHAR / TEXT / DATE / TIMESTAMP | `String` |
| nullable カラム | `Option<T>` |

## fav explain --lineage

`!Snowflake` エフェクトを持つ stage のリネージを解析すると、
`snowflake.query` / `snowflake.execute` の呼び出しに応じて
`!Snowflake(read)` / `!Snowflake(write)` が区別表示される。

```bash
fav explain --lineage src/etl.fav
```

出力例:

```
Lineage: src/etl.fav

Sources:
  - (SnowflakeInsert:snowflake-write)
  - (QuerySummary:snowflake-read)

Transformations:
  stage LoadCsv        [!Io]
  stage TransformRows  [Pure]
  stage SnowflakeInsert [!Snowflake(write)]  sinks=[(SnowflakeInsert:snowflake-write)]
  stage QuerySummary   [!Snowflake(read)]    sources=[(QuerySummary:snowflake-read)]

Pipelines:
  seq DemoPipeline = LoadCsv |> TransformRows |> SnowflakeInsert |> QuerySummary
```

## 完全なコード例

CSV ファイルを読み込み、Snowflake に INSERT してクエリ集計する ETL パイプライン:

```favnir
import rune "snowflake"
import rune "csv"
import rune "aws"

type OrderRow   = { order_id: Int  customer: String  amount: Float  region: String }
type SummaryRow = { region: String  total: Float  count: Int }

stage LoadCsv: String -> List<OrderRow> !IO = |path| {
  csv.read<OrderRow>(path)
}

stage TransformRows: List<OrderRow> -> List<OrderRow> = |rows| {
  List.filter(rows, |r| r.amount > 0.0)
}

stage SnowflakeInsert: List<OrderRow> -> Int !Snowflake = |rows| {
  bind json_str <- Json.encode_raw(rows)
  snowflake.execute(
    $"INSERT INTO DEMO_DB.PUBLIC.ORDERS SELECT value:order_id::INT, value:customer::STRING, value:amount::FLOAT, value:region::STRING FROM TABLE(FLATTEN(PARSE_JSON('{json_str}')))"
  )
}

stage QuerySummary: Int -> Unit !Snowflake !AWS = |_| {
  bind rows <- snowflake.query<SummaryRow>(
    "SELECT region, SUM(amount) AS total, COUNT(*) AS count FROM DEMO_DB.PUBLIC.ORDERS GROUP BY region ORDER BY region"
  )
  bind ts <- aws.timestamp()
  aws.s3_put_json($"my-bucket/output/summary-{ts}.json", rows)
}

seq ETLPipeline = LoadCsv |> TransformRows |> SnowflakeInsert |> QuerySummary
```

実行:

```bash
fav run src/etl.fav data/orders.csv
```
