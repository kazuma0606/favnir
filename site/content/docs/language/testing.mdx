# テスト・ベンチマーク

Favnir はテストとベンチマークを言語レベルでサポートしています。外部ライブラリ不要で、`.fav` ファイルの中に直接 `test` / `bench` ブロックを書けます。

## test ブロック

```favnir
test "description" {
    <式 — アサーション関数または Bool 式>
}
```

テストブロック内では `assert_eq` / `assert_ok` / `assert_err` / `assert_true` の 4 つのアサーション関数を使います。アサーションが失敗すると即座にテスト FAIL となり、エラーメッセージが表示されます。

### 例

```favnir
fn add(a: Int, b: Int) -> Int { a + b }

test "add returns correct result" {
    assert_eq(add(1, 2), 3)
}

test "add with negatives" {
    assert_true(add(-1, 1) == 0)
}
```

## アサーション関数（4 種類）

| 関数 | シグネチャ | 説明 |
|---|---|---|
| `assert_eq(a, b)` | `(T, T) -> Unit` | `a == b` でなければ FAIL |
| `assert_ok(r)` | `(Result<T, E>) -> T` | `Result.ok(v)` でなければ FAIL。ok なら v を返す |
| `assert_err(r)` | `(Result<T, E>) -> E` | `Result.err(e)` でなければ FAIL。err なら e を返す |
| `assert_true(b)` | `(Bool) -> Unit` | `b == true` でなければ FAIL |

### assert_ok の使用例

```favnir
test "insert returns ok" {
    bind result <- db_insert(row)
    assert_ok(result)
}
```

### assert_err の使用例

```favnir
test "invalid input returns err" {
    bind result <- validate("bad@input")
    bind msg <- assert_err(result)
    assert_eq(msg, "invalid email")
}
```

### assert_true の使用例

```favnir
test "list length is correct" {
    bind xs <- load_rows()
    bind rows <- assert_ok(xs)
    assert_true(List.length(rows) > 0)
}
```

### bind を使ったテスト

複数のステップが必要なテストには `bind` を使います。

```favnir
test "transform trims whitespace" {
    bind row <- Result.ok({ full_name: "  Alice  ", email: "A@example.com" })
    bind result <- transform_row(row)
    assert_eq(result.full_name, "Alice")
}
```

### Ctx.mock との組み合わせ

```favnir
test "pipeline with mock db" {
    bind ctx <- Ctx.mock({ db: MockDb.empty() })
    bind result <- run_pipeline(ctx)
    assert_ok(result)
}
```

## fav test コマンド

### 単一ファイルのテスト

```bash
fav test myfile.fav
```

### ディレクトリ全体のテスト

```bash
fav test src/
```

### オプション

| オプション | 説明 |
|-----------|------|
| `--filter <pattern>` | テスト名に pattern を含むものだけ実行 |
| `--fail-fast` | 最初の失敗で停止 |
| `--no-capture` | 標準出力をキャプチャせずに表示 |
| `--coverage` | カバレッジレポートを出力 |

```bash
# "list" を含むテストだけ実行
fav test src/ --filter list

# 最初の失敗で停止
fav test src/ --fail-fast
```

## テスト出力フォーマット

```
running 3 tests in myfile.fav

  PASS  add returns correct result
  PASS  add with negatives
  FAIL  safe_div returns Some when divisor is nonzero

test result: FAIL — 2 passed, 1 failed
```

テストが全件 PASS の場合:

```
running 3 tests in myfile.fav

  PASS  add returns correct result
  PASS  add with negatives
  PASS  safe_div returns None when divisor is zero

test result: ok — 3 passed, 0 failed
```

## bench ブロック

ベンチマークは `bench` キーワードで定義します。

```favnir
bench "description" {
    <繰り返し実行したい式>
}
```

### 例

```favnir
bench "list range 1000" {
    List.range(0, 1000)
}

bench "string concat 100 times" {
    List.fold(List.range(0, 100), "", |acc, _| String.concat(acc, "x"))
}
```

## fav bench コマンド

```bash
fav bench myfile.fav
```

### 出力フォーマット

```
running 2 benchmarks in myfile.fav

  list range 1000          1,234 ns/iter  (±  12 ns)
  string concat 100 times  5,678 ns/iter  (± 123 ns)
```

## テストのベストプラクティス

- テストは1つの関数・動作に絞る（1テスト1アサーション）
- テスト名は「何を確認するか」を日本語または英語で明確に
- `Result.err` や `Option.none` のケースも必ずテストする
- 境界値（0、空リスト、最大値など）をテストに含める
