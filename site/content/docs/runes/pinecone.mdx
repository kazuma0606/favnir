# Pinecone Rune

Pinecone ベクトルDB と Favnir パイプラインを統合します。
LLM Rune と組み合わせることで、ドキュメントのベクトル化・保存から近傍検索・コンテキスト取得まで
`stage` として表現できます。RAG（Retrieval-Augmented Generation）パイプラインを `fav run` で実行可能にします。

## セットアップ

```bash
# Pinecone API キーを環境変数で指定
export PINECONE_API_KEY=your-api-key
export PINECONE_BASE_URL=https://your-index-env.svc.pinecone.io
```

## クイックスタート

```favnir
import runes/pinecone
import runes/llm

// ドキュメントをベクトル化して Pinecone に保存
stage IndexDocuments: List<String> -> Unit = |docs| {
  bind vectors <- docs
    |> List.map(|doc|
      bind embedding <- LLM.embed(config.openai, doc)
      Result.ok(PineconeVector { id: Gen.uuid(), values: embedding, metadata: doc })
    )
    |> Result.all
  Pinecone.upsert(config.pinecone_index, vectors)
}

// クエリに関連するドキュメントを取得（RAG）
stage SearchDocuments: String -> List<String> = |query| {
  bind embedding <- LLM.embed(config.openai, query)
  Pinecone.query(config.pinecone_index, embedding, 5, "")
}
```

## API リファレンス

### `Pinecone.upsert`

```favnir
fn Pinecone.upsert(index: String, vectors: List<PineconeVector>) -> Result<Unit, String>
```

ベクトルをインデックスに追加・更新します（バッチ対応）。

### `Pinecone.query`

```favnir
fn Pinecone.query(index: String, vector: List<Float>, k: Int, filter: String) -> Result<List<String>, String>
```

クエリベクトルに近い上位 `k` 件を返します。`filter` にメタデータフィルタ（JSON 文字列）を指定できます。

### `Pinecone.delete`

```favnir
fn Pinecone.delete(index: String, ids: List<String>) -> Result<Unit, String>
```

ID 指定でベクトルを削除します。

### `Pinecone.fetch`

```favnir
fn Pinecone.fetch(index: String, ids: List<String>) -> Result<List<String>, String>
```

ID 指定でベクトルを取得します。

> **注意**: Pinecone API の fetch は本来 GET メソッドですが、現在は `Http.post_json`（POST）で実装しています。`Http.get_with_body` primitive の実装後に移行予定です。

### `Pinecone.describe_index_stats`

```favnir
fn Pinecone.describe_index_stats(index: String) -> Result<String, String>
```

インデックスの統計情報（ベクトル数・次元数等）を取得します。

## ヘルパー型

```favnir
type PineconeVector = {
  id: String,
  values: List<Float>,
  metadata: String
}
```

## 接続設定

| 環境変数 | デフォルト | 説明 |
|---|---|---|
| `PINECONE_API_KEY` | — | Pinecone API キー（HTTP 有効化後に `Api-Key` ヘッダーへ自動付与予定）|
| `PINECONE_ENV` | — | 環境名（例: `us-east1-gcp`）|
| `PINECONE_BASE_URL` | `https://api.pinecone.io` | エンドポイント URL |
