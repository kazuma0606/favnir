# テスト（Testing）

Favnir はテスト DSL を内蔵しています。`.fav` ファイルに直接テストを書いて `fav test` で実行できます。

---

## 基本的なテスト

```favnir
test "数の加算" {
  let result = 1 + 1
  assert_eq(result, 2)
}

test "文字列のトリム" {
  let trimmed = String.trim("  hello  ")
  assert_eq(trimmed, "hello")
}
```

実行方法:

```bash
fav test                          # 全テスト
fav test myfile.fav               # 単一ファイル
fav test --filter "加算"          # 名前でフィルタ
fav test --fail-fast              # 最初の失敗で停止
```

## アサート一覧

### assert_eq（v16.7.0）

```favnir
assert_eq(actual, expected)
// 不一致の場合: "assert_eq failed:\n  actual: X\n  expected: Y"
assert_eq(1 + 1, 2)
assert_eq(String.trim("  hi  "), "hi")
```

値は `vmvalue_repr` で文字列化して比較するため、任意の型で使えます。

### assert_approx_eq（v16.7.0）

```favnir
// 浮動小数点の近似比較: |actual - expected| <= epsilon
assert_approx_eq(Math.sqrt(2.0), 1.414, 0.001)
```

### assert_true / assert_ok / assert_err（v15.3.0）

```favnir
assert_true(condition)        // Bool が false なら FAIL
assert_ok(result)             // Result が err なら FAIL
assert_err(result)            // Result が ok なら FAIL
```

### assert_contains / assert_length（v16.7.0）

```favnir
let rows = [1, 2, 3]
assert_contains(rows, 2)      // 要素が存在しなければ FAIL
assert_length(rows, 3)        // 長さが一致しなければ FAIL
```

### assert_str_contains / assert_str_starts_with（v16.7.0）

```favnir
assert_str_contains("Hello, World!", "World")
assert_str_starts_with("Hello, World!", "Hello")
```

### assert_err_eq（v16.7.0）

```favnir
// エラーメッセージの内容を検証
let result = Result.err("invalid email")
assert_err_eq(result, "invalid email")
```

### assert_snapshot（v16.7.0）

```favnir
assert_snapshot(value, "snapshot_name")
```

- **初回実行**: `.snap/snapshot_name.snap` にスナップショットを保存してパス
- **2 回目以降**: 保存済みスナップショットと比較。一致 → パス、不一致 → FAIL
- **スナップショット更新**: `fav test --update-snapshots` で全スナップショットを上書き

`.snap/` ディレクトリをプロジェクトルートに自動作成します。`.gitignore` に追加するか、コミットしてリグレッションテストに活用できます。

## test_group でテストをグループ化（v16.7.0）

関連するテストをグループ化してサマリーをグループ単位で出力できます:

```favnir
test_group "変換パイプライン" {
  test "空白をトリムする" {
    let result = String.trim("  Alice  ")
    assert_eq(result, "Alice")
  }

  test "空入力を拒否する" {
    let result = Result.err("empty")
    assert_err(result)
  }

  test "メールを含むか確認" {
    assert_str_contains("alice@example.com", "@")
  }
}
```

出力例:
```
running 3 tests in "変換パイプライン"

  test 空白をトリムする   ... PASS
  test 空入力を拒否する   ... PASS
  test メールを含むか確認 ... PASS

test result: ok. 3 passed; 0 failed; 0 filtered; finished in 2ms
```

## fav test フラグ

| フラグ | 説明 |
|---|---|
| `--filter <pattern>` | 名前にパターンを含むテストのみ実行 |
| `--fail-fast` | 最初の失敗で停止 |
| `--no-capture` | テスト中の IO 出力を表示（デフォルト: 抑制） |
| `--update-snapshots` | 全スナップショットを上書き更新 |
| `--coverage` | カバレッジ計測を有効化 |
| `--coverage-report <dir>` | HTML カバレッジレポートをディレクトリに出力 |

## パターン: let バインディングでセットアップ

テストボディは `let` / `bind` バインディングの後に単一のアサーション式を置く形式です:

```favnir
test "パイプラインが行を処理する" {
  let rows = [
    { name: "  Alice  ", score: 95 },
    { name: "Bob", score: 87 },
  ]
  let count = List.length(rows)
  assert_eq(count, 2)
}
```

## パターン: エラーパスのテスト

```favnir
test "パースエラーは err を返す" {
  let result = DateTime.parse("not a date")
  assert_err(result)
}

test "エラーメッセージが一致する" {
  let result = Result.err("invalid email")
  assert_err_eq(result, "invalid email")
}
```

## bench（ベンチマーク）

```favnir
bench "List.sort のベンチマーク" {
  let data = [3, 1, 4, 1, 5, 9, 2, 6]
  List.sort_by(data, |x| x)
}
```

```bash
fav bench                        # 全ベンチマーク実行
fav bench --filter "sort"
```
