# Favnir v3.4.0 Language Specification

## Overview

Favnir v3.4.0 adds the `fav infer` command for automatic type definition generation
from CSV files and DB table schemas. No language primitives, types, or effects are added —
this is a purely toolchain-level addition.

---

## `fav infer` Command

### Synopsis

```bash
fav infer [csv_path] [table_name] [--db <conn_str>] [--out <path>] [--name <TypeName>]
```

### Flags

| Flag | Description | Default |
|------|-------------|---------|
| `--db <conn_str>` | DB connection string (SQLite or PostgreSQL) | — |
| `--out <path>` | Output file path; use trailing `/` for directory | stdout |
| `--name <TypeName>` | Type name for CSV inference | `Row` |

---

## CSV Inference

```bash
fav infer data.csv
fav infer data.csv --name MyRecord --out types.fav
```

Reads the CSV header row and up to 100 data rows, infers one `type` definition.

### Example

Input `data.csv`:
```
id,name,value,region,notes
1,Alice,3.14,EU,
2,Bob,2.71,US,hello
```

Output:
```favnir
// auto-generated by `fav infer data.csv`
// Review and adjust before use.
type Row = {
    id:     Int
    name:   String
    value:  Float
    region: Option<String>
    notes:  Option<String>
}
```

### Type Inference Rules

| Sample values | Inferred type |
|---------------|---------------|
| `1`, `42`, `-3` | `Int` |
| `3.14`, `2.0`, `-1.5` | `Float` |
| `true`, `false` | `Bool` |
| Other strings | `String` |
| Any empty value in column | `Option<T>` |
| All values empty | `Option<String>` |

Priority: `Bool` > `Int` > `Float` > `String`

## DB Schema Inference

### SQLite

```bash
fav infer --db sqlite:path/to/db.sqlite
fav infer --db sqlite:path/to/db.sqlite users
fav infer --db sqlite::memory: events --out schema/events.fav
```

Uses `PRAGMA table_info(table)`:
- `notnull = 0` → `Option<T>`
- Table name → PascalCase type name (`users` → `User`, `user_profiles` → `UserProfile`)

### PostgreSQL

```bash
fav infer --db postgres://user:pass@localhost/mydb
fav infer --db postgres://user:pass@localhost/mydb users --out schema/users.fav
```

Requires `--features postgres_integration` at build time.
Uses `information_schema.columns`:
- `is_nullable = 'YES'` → `Option<T>`

### DB Column Type Mapping

**SQLite**:

| SQLite type | Favnir type |
|-------------|-------------|
| `INTEGER`, `INT`, `BIGINT` | `Int` |
| `REAL`, `FLOAT`, `DOUBLE`, `NUMERIC`, `DECIMAL` | `Float` |
| `BOOLEAN`, `BOOL` | `Bool` |
| Everything else | `String` |

**PostgreSQL**:

| PostgreSQL type | Favnir type |
|-----------------|-------------|
| `integer`, `bigint`, `smallint`, `serial` | `Int` |
| `real`, `double precision`, `numeric`, `decimal` | `Float` |
| `boolean` | `Bool` |
| Everything else | `String` |

## `--out` Flag

| Value | Behavior |
|-------|----------|
| omitted | Print to stdout |
| `types.fav` | Write all definitions to single file |
| `schema/` (trailing slash) | Write one `.fav` file per table/type into the directory |

For directory output, file names are lowercased type names: `User` → `user.fav`.

---

## Type Name Derivation

For `fav infer --db`:
- Table name is converted to PascalCase
- Trailing `s` is stripped (simple heuristic):
  - `users` → `User`
  - `orders` → `Order`
  - `user_profiles` → `UserProfile`
  - `events` → `Event`

For `fav infer <csv>`:
- Default: `Row`
- Override with `--name <TypeName>`

---

## Generated Comment Header

All generated files include:

```favnir
// auto-generated by `fav infer <source>`
// Review and adjust before use.
```

This signals that the type definitions are a starting draft for human review.
The generated types are directly usable with `Schema.adapt<T>` (v3.2.0+).

---

## Error Cases

| Situation | Behavior |
|-----------|----------|
| CSV file not found | `error: CSV file not found: <path>` → exit 1 |
| CSV has no header row | `error: CSV has no header row` → exit 1 |
| DB connection failed | `error: DB connection failed: <message>` → exit 1 |
| Table not found | `error: table '<name>' not found in database` → exit 1 |
| `--out` directory not creatable | `error: cannot create output directory` → exit 1 |
| `postgres://` without feature | `error: postgres:// requires --features postgres_integration` → exit 1 |

## Usage with `Schema.adapt`

The generated type definitions are directly compatible with `Schema.adapt<T>`:

```favnir
// schema/user.fav (generated by fav infer --db sqlite:app.sqlite users)
type User = {
    id:   Int
    name: String
    age:  Option<Int>
}
```

```favnir
import rune "db"

public fn main() -> Unit !Io !Db {
    chain conn  <- db.connect(Env.get_or("DB_URL", "sqlite:app.sqlite"))
    bind  raw   <- DB.query_raw(conn, "SELECT id, name, age FROM users")
    chain users <- Schema.adapt(raw, "User")
    for u in users { IO.println($"{u.id}: {u.name}"); }
}
```
