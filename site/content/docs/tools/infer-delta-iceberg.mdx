# fav infer --from delta / --from iceberg

`fav infer` コマンドの `--from delta` / `--from iceberg` サブコマンドで、
Delta Lake / Apache Iceberg テーブルのスキーマから Favnir 型定義を自動生成する。

> **v27.7.0 stub**: 実際のテーブルスキーマ読み取りは v28.x（`delta-rs` / `iceberg-rust` 統合時）に実装予定。
> 現バージョンでは固定の 5 フィールドスキーマを出力する。

## 使用例

```bash
# Delta Lake テーブルからスキーマを自動生成
fav infer --from delta --path /tmp/orders

# S3 上の Delta テーブル
fav infer --from delta --path s3://bucket/my_table

# Iceberg テーブルから（REST カタログ）
fav infer --from iceberg --catalog http://localhost:8181 --table warehouse.orders

# ファイルに出力
fav infer --from delta --path /tmp/orders --out schema.fav
```

## 出力例

```favnir
// Generated by fav infer --from delta --path /tmp/orders
// NOTE: v27.7.0 stub — actual schema requires delta-rs integration (v28.x)
type OrdersRow = {
  id: Int,
  name: String,
  amount: Float,
  created_at: DateTime,
  active: Bool,
}
```

## 型名の生成ルール

パスまたはテーブル名の末尾セグメントから PascalCase の型名を生成し、`Row` を付加する。

| 入力 | 生成される型名 |
|---|---|
| `/tmp/orders` | `OrdersRow` |
| `s3://bucket/my_table` | `MyTableRow` |
| `warehouse.orders` | `OrdersRow` |

## 型マッピング

| Delta / Iceberg 型 | Favnir 型 |
|---|---|
| `long` / `integer` / `int` | `Int` |
| `double` / `float` | `Float` |
| `string` | `String` |
| `timestamp` / `timestamp_ntz` | `DateTime` |
| `boolean` | `Bool` |
| その他 / 不明 | `String`（デフォルト） |

## フラグ

| フラグ | 説明 | 必須 |
|---|---|---|
| `--from delta` | Delta Lake モードを選択 | — |
| `--from iceberg` | Apache Iceberg モードを選択 | — |
| `--path <path>` | Delta テーブルのパス（`--from delta` 時必須） | delta のみ |
| `--catalog <url>` | Iceberg REST カタログ URL（`--from iceberg` 時必須） | iceberg のみ |
| `--table <name>` | テーブル名（`--from iceberg` 時必須） | iceberg のみ |
| `--out <file>` | 出力ファイルパス（省略時は標準出力） | 省略可 |

## v28.x 以降の予定

- `delta-rs` クレートを使った実 Delta Lake スキーマ読み取り
- `iceberg-rust` クレートを使った実 Iceberg スキーマ読み取り
- Nested 型（`struct` / `array` / `map`）の Favnir 型マッピング
- `null` / `nullable` フィールドの `Option<T>` マッピング
