# インクリメンタルコンパイル

v19.3.0 から、`fav build` はビルドキャッシュを使用して変更のないファイルのコンパイルをスキップします。

## 速度の改善

| シナリオ | v19.2.0 以前 | v19.3.0 以降 |
|---|---|---|
| 初回ビルド（50 ファイル） | 10.0 秒 | 10.0 秒（同じ） |
| 変更なし 2 回目 | 10.0 秒 | 0.3 秒 |
| 1 ファイル変更 | 10.0 秒 | 1.2 秒 |

## キャッシュの仕組み

```
.fav → ファイルコンテンツの SHA-256 を計算
     → ~/.fav/cache/<project-hash>/<file-hash>.ir を確認
     → キャッシュがあれば読み込んでスキップ
     → なければコンパイルしてキャッシュに保存
```

## 依存グラフ追跡

```
use utils.format_date     # pipeline.fav → utils.fav に依存
use utils.validate        # pipeline.fav → utils.fav に依存

変更検出:
  utils.fav が変更 → utils.fav と pipeline.fav を再コンパイル
  types.fav が変更 → types.fav のみ再コンパイル（pipeline に影響なし）
```

## CLI オプション

```bash
# 通常ビルド（キャッシュ使用）
fav build src/pipeline.fav

# キャッシュを無視して完全再ビルド
FAV_NO_CACHE=1 fav build src/pipeline.fav

# キャッシュヒット / ミスの詳細を表示
FAV_EXPLAIN_CACHE=1 fav build src/pipeline.fav
# 出力例:
# [cache MISS] src/pipeline.fav  (a3f8b2c1...)
# [cache HIT]  src/utils.fav     (7d4e91bc...)
```

## 環境変数

| 変数 | 説明 |
|---|---|
| `FAV_NO_CACHE=1` | キャッシュを無効化して常に完全再ビルド |
| `FAV_EXPLAIN_CACHE=1` | キャッシュヒット / ミスを stderr に出力 |
| `FAV_CACHE_DIR=<path>` | キャッシュディレクトリを上書き（デフォルト: `~/.fav/cache/`） |

## キャッシュのクリア

```bash
# 全キャッシュを削除
rm -rf ~/.fav/cache/

# 特定プロジェクトのキャッシュを削除（プロジェクトルートで実行）
FAV_NO_CACHE=1 fav build src/pipeline.fav
```

## 制限事項（v19.3.0）

- キャッシュの粒度は**ファイル単位**（関数単位は v19.5 以降）
- `fav run` はキャッシュを使用しない（`fav build` → `fav run --precompiled` の組み合わせ）
- クロスコンパイル（`--target native`）はキャッシュ未対応
