# Secret Manager / Vault 連携

`runes/secret` が提供する Secret Rune を使って、AWS Secrets Manager・HashiCorp Vault・GCP Secret Manager からシークレットを取得します。
ローカル開発では `Secret.get_env` を使って環境変数からフォールバックします。

## コード例

```favnir
import runes/secret

fn connect_to_db(ctx: AppCtx) -> Result<String, String> !Http {
    bind password <- Secret.get_aws(ctx, "prod/db/password")
    Result.ok(String.concat(["postgresql://user:", password, "@db:5432/app"]))
}

fn connect_local(ctx: AppCtx) -> Result<String, String> {
    bind password <- Secret.get_env(ctx, "DB_PASSWORD")
    Result.ok(String.concat(["postgresql://user:", password, "@localhost:5432/app"]))
}
```

## `fav.toml` 設定例

```toml
[secrets]
backend = "aws"    # "aws" | "vault" | "gcp" | "env"
```

## ポイント

- `Secret.get_aws` / `Secret.get_vault` / `Secret.get_gcp` は `!Http` エフェクトを持ちます。呼び出し元の関数も `!Http` を宣言する必要があります。
- `Secret.get_env` はネットワーク通信を行わないためエフェクトなしです。ローカル開発・テスト環境でのフォールバックに適しています。
- シークレット名はコードに直接書いても安全です（値ではなく名前のため）。値は実行時にバックエンドから取得されます。
- `fav.toml` の `[secrets] backend` を切り替えることで、コード変更なしに環境を切り替えられます。
