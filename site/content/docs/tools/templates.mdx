# テンプレートギャラリー

`fav new --template <name> <project-name>` でユースケース別のスターターを生成できます。

## 使い方

```bash
fav new --template etl-csv-to-db    myetl       # CSV → DB ETL
fav new --template api-gateway      myapi        # HTTP API
fav new --template lambda-scheduled myjob        # スケジューラ
fav new --template distributed-etl  mybigproject # 分散 ETL
```

## テンプレート一覧

| テンプレート名 | 説明 | 生成される主ファイル |
|---|---|---|
| `etl-csv-to-db` | CSV → DB ETL パイプライン | `pipeline.fav` |
| `api-gateway` | HTTP API ゲートウェイ | `api.fav` |
| `lambda-scheduled` | スケジュール実行 Lambda ジョブ | `job.fav` |
| `distributed-etl` | 分散並列 ETL パイプライン | `pipeline.fav` |

各テンプレートには以下が含まれます:

- `.fav` ソースファイル（動作するサンプル）
- `fav.toml`（適切なデフォルト設定・Rune 依存）
- `README.md`（セットアップ・実行手順）
- `.github/workflows/ci.yml`（GitHub Actions CI 設定）

## etl-csv-to-db

CSV ファイルを読み込み、PostgreSQL に挿入するシンプルな ETL です。

```bash
fav new --template etl-csv-to-db myetl
cd myetl
DATABASE_URL=postgres://localhost/mydb fav run pipeline.fav
```

生成される `pipeline.fav`:

```favnir
import csv
import postgres as db

stage LoadCsv -> List[String] !Io {
    csv.read_file("data.csv")
}

stage InsertRows(rows: List[String]) -> Int !Db {
    rows |> List.map(|row| db.execute("INSERT INTO records (data) VALUES ($1)", [row]))
         |> List.length
}

pipeline myetl {
    LoadCsv |> InsertRows
}
```

## api-gateway

HTTP API エンドポイントを実装するテンプレートです。

```bash
fav new --template api-gateway myapi
cd myapi
fav run api.fav
```

## lambda-scheduled

定期実行ジョブのテンプレートです。GitHub Actions の `schedule` トリガー付き。

```bash
fav new --template lambda-scheduled myjob
cd myjob
fav run job.fav
```

## distributed-etl

`par [A, B] |> Merge` パターンで複数データソースを並列取得・結合します。

```bash
fav new --template distributed-etl mybigproject
cd mybigproject
DATABASE_URL=postgres://localhost/mydb fav run pipeline.fav
```

## 既存テンプレート

v24.8.0 以前から利用可能なテンプレートも引き続き使えます:

| テンプレート名 | 説明 |
|---|---|
| `script` | シンプルなスクリプト（デフォルト） |
| `pipeline` | 基本パイプライン |
| `lib` | ライブラリプロジェクト |
| `postgres-etl` | PostgreSQL ETL（詳細設定付き） |
