# Favnir v11.1.0 仕様書

Date: 2026-06-06
Theme: emit_python 基盤 — AST → Python コード生成の土台

---

## 概要

`fav transpile --target python <file.fav>` コマンドを新設し、
Fav の AST を Python ソースコードに変換する `emit_python.rs` を実装する。

v11.1.0 のスコープ:
1. **型定義** — `type Foo = { ... }` → `@dataclass class Foo`
2. **関数定義** — `fn foo(...) -> T` → `def foo(...) -> T`
3. **基本式** — リテラル / 演算 / `if-else` / `List.*` / `String.*`
4. **bind 脱糖** — `bind x <- expr` → `x = expr`
5. **match 変換** — `Option` / `Result` パターン
6. **CLI** — `fav transpile --target python <file>` エントリ

stage / seq / エフェクト変換は v11.2.0〜v11.4.0 で対応する。

---

## アーキテクチャ

```
.fav ソース
    ↓ frontend/parser.rs  (既存)
Program (AST)
    ↓ emit_python.rs      (新規)
Python ソース文字列
    ↓ fs::write
<output>.py
```

`emit_python.rs` は型チェック済み AST を必要とせず、
構文木のみから Python を生成する（v11.1.0 時点）。
v11.8.0 で checker.fav との統合を行う。

| Fav 型 | Python 型 | 備考 |
|---|---|---|
| `String` | `str` | |
| `Int` | `int` | |
| `Float` | `float` | |
| `Bool` | `bool` | |
| `Unit` | `None` | 戻り型のみ |
| `List<T>` | `List[T]` | `from typing import List` |
| `Option<T>` | `Optional[T]` | `from typing import Optional` |
| `Result<T, E>` | `Result` | ファイル先頭にヘルパークラス埋め込み |
| カスタム型 `Foo` | `Foo` | dataclass として定義済み前提 |

## 構文変換仕様

### 1. 型定義 — `type`

**入力 (Fav)**:
```fav
type TxnRow = {
  transaction_id: String
  amount:         Float
  region:         String
}
```

**出力 (Python)**:
```python
@dataclass
class TxnRow:
    transaction_id: str
    amount: float
    region: str
```

Sum type（`type Shape = Circle(Float) | Rect(Float, Float)` 等）は v11.1.0 スコープ外。
名目型ラッパー（`type Name(Inner)`）も v11.1.0 スコープ外。

---

### 2. 関数定義 — `fn`

**入力 (Fav)**:
```fav
fn add(a: Int, b: Int) -> Int {
  a + b
}

fn greet(name: String) -> Unit !IO {
  IO.println(String.concat("Hello, ", name))
}
```

**出力 (Python)**:
```python
def add(a: int, b: int) -> int:
    return a + b

# effects: IO
def greet(name: str) -> None:
    print("Hello, " + name)
```

エフェクト宣言はコメントに落とす（v11.3.0 以降で実際の変換を行う）。
`!IO` の `IO.println` は v11.3.0 で対応。v11.1.0 では `IO.*` 呼び出しは
`_io_<name>(...)` のプレースホルダーとして出力する。

---

### 3. `bind` 脱糖

**入力 (Fav)**:
```fav
fn process(x: Int) -> Int {
  bind a <- double(x)
  bind b <- triple(a)
  a + b
}
```

**出力 (Python)**:
```python
def process(x: int) -> int:
    a = double(x)
    b = triple(a)
    return a + b
```

ブロック末尾の式は `return` 付きで出力する。
`bind _ <- expr` は `expr` のみ（変数束縛なし）。

---

### 4. `if-else` 式

**入力 (Fav)**:
```fav
if x > 0 { "positive" }
else { "non-positive" }
```

**出力 (Python)**:
```python
("positive" if x > 0 else "non-positive")
```

ブロック内に複数ステートメントがある場合は三項式では表現できないため、
インライン関数（`_cond_<n>`）として切り出す:

```python
def _cond_0():
    if x > 0:
        return "positive"
    else:
        return "non-positive"
_cond_0()
```

---

### 5. `match` — Option パターン

**入力 (Fav)**:
```fav
match List.first(lst) {
  Some(v) => v
  None    => ""
}
```

**出力 (Python)**:
```python
(lambda _m: _m if _m is not None else "")(lst[0] if lst else None)
```

複数ステートメントを含むアームの場合は `_match_<n>` ヘルパー関数に切り出す。

---

### 6. `match` — Result パターン

**入力 (Fav)**:
```fav
match IO.read_file_raw(path) {
  Ok(text) => text
  Err(e)   => ""
}
```

**出力 (Python)**:
```python
# v11.1.0: IO.read_file_raw → プレースホルダー（v11.3.0 で実変換）
_r = _io_read_file_raw(path)
if isinstance(_r, Ok):
    text = _r.value
    _result = text
elif isinstance(_r, Err):
    e = _r.error
    _result = ""
_result
```

---

### 7. 標準ライブラリ変換（v11.1.0 対応分）

| Fav 呼び出し | Python 変換 |
|---|---|
| `List.empty()` | `[]` |
| `List.concat(a, b)` | `a + b` |
| `List.length(lst)` | `len(lst)` |
| `List.filter(lst, f)` | `[_x for _x in lst if f(_x)]` |
| `List.map(lst, f)` | `[f(_x) for _x in lst]` |
| `List.first(lst)` | `(lst[0] if lst else None)` |
| `List.drop(lst, n)` | `lst[n:]` |
| `List.nth(lst, n)` | `(lst[n] if len(lst) > n else None)` |
| `String.concat(a, b)` | `a + b` |
| `Int.to_string(n)` | `str(n)` |
| `Float.to_string(f)` | `str(f)` |
| `Bool.to_string(b)` | `str(b).lower()` |

---

### 8. ファイル先頭の埋め込みヘルパー

生成 Python ファイルの先頭には常に以下を出力する:

```python
# Generated by fav transpile --target python
# Source: <source_file>
from __future__ import annotations
from dataclasses import dataclass, asdict
from typing import List, Optional, Any
import sys

class Ok:
    def __init__(self, value): self.value = value
class Err:
    def __init__(self, error): self.error = error
```

## CLI 仕様

```
fav transpile --target <lang> <file.fav> [--out <output>]
```

| オプション | 説明 | デフォルト |
|---|---|---|
| `--target python` | 出力言語（v11.1.0 は python のみ） | 必須 |
| `--out <file>` | 出力ファイルパス | `<input>.py`（拡張子置換） |

エラーケース:
- `--target` 未指定 → `error: --target is required`
- `.fav` ファイルが存在しない → `error: file not found: <path>`
- `--target` に未対応言語 → `error: unsupported target: <lang> (supported: python)`

## テスト方針

`emit_python.rs` 内の `#[cfg(test)] mod v11100_tests` に 7 件追加。
すべて「Fav ソース文字列 → parse → emit_python → Python 文字列に期待値を含む」形式。

```rust
#[test]
fn transpile_dataclass_simple() {
    let src = "type Point = { x: Int  y: Int }";
    let out = emit_python_str(src);
    assert!(out.contains("@dataclass"));
    assert!(out.contains("class Point:"));
    assert!(out.contains("x: int"));
    assert!(out.contains("y: int"));
}
```
