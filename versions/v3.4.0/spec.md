# Favnir v3.4.0 Specification

## Theme: `fav infer` — スキーマ自動生成

v3.4.0 は既存の CSV ファイルおよび DB テーブルから Favnir の型定義を自動生成する
`fav infer` コマンドを追加する。

パイプラインを書く際の「最初の型定義を手書きする」というコストを排除し、
既存データからすぐに始められるようにする。

生成された型定義はあくまで草案（draft）であり、
コメントに `// Review and adjust before use.` を付与する。

---

## 1. CLI インターフェース

```bash
# CSV から型推論
fav infer data.csv
fav infer data.csv --out types.fav
fav infer data.csv --name MyRow

# SQLite から型推論
fav infer --db sqlite:path/to/db.sqlite
fav infer --db sqlite:path/to/db.sqlite users
fav infer --db sqlite::memory: events --out schema/events.fav

# PostgreSQL から型推論（postgres_integration feature が必要）
fav infer --db postgres://user:pass@localhost/mydb
fav infer --db postgres://user:pass@localhost/mydb users --out schema/users.fav

# 複数テーブルをディレクトリに出力
fav infer --db sqlite:app.sqlite --out schema/
# → schema/users.fav, schema/orders.fav, ... を生成
```

### フラグ一覧

| フラグ | 説明 | デフォルト |
|--------|------|----------|
| `--db <conn_str>` | DB 接続文字列（指定時は CSV パスの代わりに DB を使用） | なし |
| `--out <path>` | 出力先ファイルまたはディレクトリ | stdout |
| `--name <TypeName>` | CSV 型定義の型名（CSV のみ有効） | `Row` |

## 2. CSV からの型推論

### 2.1 入力例

```csv
id,name,value,region,notes
1,Alice,3.14,EU,
2,Bob,2.71,US,hello
3,Carol,1.41,,
```

### 2.2 出力例

```bash
$ fav infer data.csv
```

```favnir
// auto-generated by `fav infer data.csv`
// Review and adjust before use.
type Row = {
    id:     Int
    name:   String
    value:  Float
    region: Option<String>
    notes:  Option<String>
}
```

### 2.3 型推論ルール

| サンプル値 | 推論される型 |
|-----------|-------------|
| `1`, `42`, `-3` | `Int` |
| `3.14`, `2.0`, `-1.5` | `Float` |
| `true`, `false` | `Bool` |
| `2026-01-01`, `2026-01-01T00:00:00` | `String`（Date/DateTime は将来候補） |
| 上記以外の文字列 | `String` |
| 1 つでも空値がある | `Option<T>` |
| 全行が空値 | `Option<String>` |

**優先度**: `Bool` > `Int` > `Float` > `String`

カラム内の全非空値が Bool に一致すれば `Bool`、
そうでなく全非空値が整数パターンなら `Int`、
そうでなく全非空値が浮動小数点パターンなら `Float`、
それ以外は `String`。

**最小サンプル行数**: ヘッダー行を除いた最初の最大 100 行を読んで推論する。
空ファイル（ヘッダーのみ）の場合は全カラムを `String` にフォールバックする。

### 2.4 型名の決定

- `--name` フラグ未指定時: `Row`
- `--name MyRecord` 指定時: `MyRecord`
- ファイル名からの自動決定はしない（`Row` が常に安全なデフォルト）

## 3. DB テーブルからの型推論

### 3.1 SQLite

SQLite `PRAGMA table_info(table_name)` を使用:

```
cid | name       | type    | notnull | dflt_value | pk
----+------------+---------+---------+------------+---
0   | id         | INTEGER | 1       | NULL       | 1
1   | name       | TEXT    | 1       | NULL       | 0
2   | email      | TEXT    | 1       | NULL       | 0
3   | created_at | TEXT    | 1       | NULL       | 0
4   | deleted_at | TEXT    | 0       | NULL       | 0
```

`notnull = 0` のカラムは `Option<T>` になる。

### 3.2 DB 型 → Favnir 型マッピング

**SQLite**:

| SQLite 型 | Favnir 型 |
|----------|----------|
| `INTEGER`, `INT`, `BIGINT`, `SMALLINT` | `Int` |
| `REAL`, `FLOAT`, `DOUBLE`, `NUMERIC`, `DECIMAL` | `Float` |
| `TEXT`, `VARCHAR`, `CHAR`, `CLOB` | `String` |
| `BLOB` | `String`（バイナリはまだ未対応） |
| `BOOLEAN` | `Bool` |
| `DATE`, `DATETIME`, `TIMESTAMP` | `String`（Date 型は将来候補） |
| その他 / 認識不能 | `String` |

**PostgreSQL**（`information_schema.columns` を使用）:

| PostgreSQL 型 | Favnir 型 |
|--------------|----------|
| `integer`, `bigint`, `smallint`, `serial` | `Int` |
| `real`, `double precision`, `numeric`, `decimal` | `Float` |
| `text`, `varchar`, `char`, `name` | `String` |
| `boolean` | `Bool` |
| `date`, `timestamp`, `timestamptz` | `String` |
| `bytea` | `String` |
| その他 | `String` |

`is_nullable = 'YES'` の場合 → `Option<T>`

### 3.3 型名の決定

- テーブル名を PascalCase に変換:
  - `users` → `User`（末尾 `s` を落とす）
  - `orders` → `Order`
  - `user_profiles` → `UserProfile`
  - `events` → `Event`
  - `data` → `Data`（末尾 `s` なし → そのまま PascalCase）
- 末尾 `s` の除去はシンプルな文字列処理のみ（複数形の一般化はしない）

### 3.4 全テーブル推論

`--db conn_str` のみ指定（テーブル名なし）の場合:

- **SQLite**: `SELECT name FROM sqlite_master WHERE type='table'` でテーブル一覧を取得
- **PostgreSQL**: `information_schema.tables WHERE table_schema='public'` でテーブル一覧を取得

`--out schema/` でディレクトリ指定の場合は `schema/<table>.fav` を生成。
`--out` なし（stdout）の場合は全テーブルを連結して出力する。

## 4. 出力フォーマット

### 4.1 生成コメント

全生成ファイルの先頭に付与:

```favnir
// auto-generated by `fav infer <source>`
// Review and adjust before use.
```

### 4.2 フィールドの整列

フィールド名の最大長にあわせて型をスペース整列する:

```favnir
type User = {
    id:         Int
    name:       String
    email:      String
    created_at: String
    deleted_at: Option<String>
}
```

### 4.3 複数型定義（全テーブル）

stdout に連続出力する場合は空行で区切る:

```favnir
// auto-generated by `fav infer --db sqlite:app.sqlite`
// Review and adjust before use.

type User = {
    id:   Int
    name: String
}

type Order = {
    id:      Int
    user_id: Int
    amount:  Float
}
```

## 5. エラー処理

| 状況 | 挙動 |
|------|------|
| CSV ファイルが存在しない | stderr にエラー → exit 1 |
| CSV のヘッダー行がない | stderr にエラー → exit 1 |
| DB 接続失敗 | stderr に接続エラー → exit 1 |
| テーブルが存在しない | stderr にエラー → exit 1 |
| `--out` のディレクトリが作成できない | stderr にエラー → exit 1 |
| PostgreSQL feature なしで postgres:// | stderr に「postgres_integration feature が必要」 → exit 1 |

`fav infer` はランタイムエラーコード（E0xxx）を出さない。
推論自体は失敗しない（型が曖昧な場合は `String` にフォールバック）。

## 6. 生成型定義の利用

生成した型定義は `Schema.adapt<T>` とそのまま組み合わせて使える:

```favnir
// auto-generated by `fav infer --db sqlite:app.sqlite users`
type User = {
    id:   Int
    name: String
    age:  Option<Int>
}
```

```favnir
import rune "db"

public fn main() -> Unit !Io !Db {
    chain conn  <- db.connect(Env.get_or("DB_URL", "sqlite:app.sqlite"))
    bind  raw   <- DB.query_raw(conn, "SELECT id, name, age FROM users")
    chain users <- Schema.adapt<User>(raw)
    for u in users { IO.println($"{u.id}: {u.name}"); }
}
```

## 7. 非ゴール（v3.4.0 スコープ外）

- `Date` / `DateTime` 型への推論（将来: `type Date = { ... }` or opaque）
- JSON カラムのネスト型展開
- `fav infer --proto`（Protocol Buffer スキーマ生成 → v3.8.0）
- MySQL 対応
- 推論設定ファイル（`.favnir-infer.toml`）
- カラム名の snake_case → camelCase 変換オプション

---

## 8. 完了条件

- `fav infer data.csv` でヘッダーと値から型定義を stdout に出力できる
- `fav infer --db sqlite:path` でテーブルスキーマから型定義を生成できる
- nullable カラム / 空値のあるカラムが `Option<T>` になる
- `--out <file>` でファイルに書き出せる
- `--out <dir>/` でテーブルごとにファイルを分けて書き出せる
- 生成した型定義で `Schema.adapt<T>` がそのまま動く（既存テストで確認）
- 既存テストが全て通る（706 → ≥ 730 tests passing 目安）
