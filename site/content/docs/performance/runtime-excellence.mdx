## 最適化の内訳

### スーパー命令（v20.2.0） {#superinstruction}

top-10 opcode ペアを融合（`LoadAdd` / `AddStore` / `LoadCmp` / `CmpJump` 等）。
1 回のディスパッチで複数命令を処理することで `tight_loop_10m_iter` が +25% 改善。

```
改善前: LoadLocal(0) → Add → StoreLocal(1)  // 3 回ディスパッチ
改善後: AddLocalLocal(0, 1)                  // 1 回ディスパッチ
```

### NaN-boxing（v20.3.0） {#nan-boxing}

詳細は [NaN-boxing](./nan-boxing) を参照。

VMValue を 64 bit 幅に圧縮。`Vec<VMValue>` のメモリフットプリントが 1/4 以下になり
キャッシュヒット率が大幅改善。`record_transform_1m` が +2x、`cold_start` が -10ms。

### DuckDB プッシュダウン（v20.4.0） {#pushdown}

詳細は [DuckDB プッシュダウン](./pushdown) を参照。

`fav explain --lineage` の静的解析を活用し、`List.filter` / `List.group_by` / `List.sum_by` 等を
DuckDB SQL に自動変換して委譲。集計クエリが +10〜100x 改善。

### mmap + SIMD CSV パーサー（v20.5.0） {#mmap-csv}

`memmap2` でファイルをゼロコピーマッピングし、`arrow-csv` の SIMD パーサーで処理。
`csv_10gb_throughput` が 340 MB/s → 1.2 GB/s（+4x）、ピークメモリ -40%。

### io_uring 非同期 I/O（v20.6.0） {#io-uring}

Linux 5.1+ 環境では `tokio-uring` によるリング I/O で syscall 削減。
Windows / macOS では epoll / kqueue へ自動フォールバック。

### Arena アロケータ（v20.7.0） {#arena}

1 chunk（1000 行）= 1 arena。chunk 完了時に一括解放（`bumpalo::Bump`）。
ストリーミングパイプラインの定常メモリ -20%、malloc/free ラウンドトリップ削減。

### Postgres コネクションプール（v20.8.0） {#connection-pool}

詳細は [Postgres Rune](../runes/postgres) を参照。

`Postgres.Pool.create/query/execute/stats/close` によるプール管理。
接続確立コスト（~50ms/回）を排除し、多段パイプラインの DB アクセスを大幅に高速化。

## ベンチマーク基盤

計測スイート（`benchmarks/suite/`）と CI 統合（`.github/workflows/bench.yml`）は v20.1.0 で整備済み。

```bash
# ベンチマーク全実行
bash benchmarks/suite/run_all.sh --format json > benchmarks/latest.json

# v20.0.0 ベースラインと比較（10% 劣化で警告）
fav run benchmarks/compare.fav -- \
  --baseline benchmarks/v20.0.0.json \
  --current  benchmarks/latest.json \
  --threshold 10
```
