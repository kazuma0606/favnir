# ADR-005: `fav` 単一バイナリにすべての機能を統合する

- **状態**: 採用
- **決定日**: v7.6.0（`cli.fav` でセルフホスト CLI 実装時）

---

## 背景

言語ツールチェーンを設計するにあたり、
「`fav-run` / `fav-check` / `fav-fmt` のように別バイナリに分割する」か
「`fav run` / `fav check` / `fav fmt` のように単一バイナリのサブコマンドにする」かを決定する必要があった。

## 決定

`fav` 単一バイナリに run / check / fmt / lint / doc / repl / watch /
profile / explain / new 等すべての機能を統合する。

## 現在のサブコマンド一覧

| コマンド | 機能 | 導入 |
|---|---|---|
| `fav run` | パイプライン実行 | v4.x |
| `fav check` | 型チェックのみ | v8.1.0 |
| `fav fmt` | フォーマッター | v9.2.0 |
| `fav lint` | 静的解析 | v9.3.0 |
| `fav doc` | ドキュメント生成 | v9.8.0 |
| `fav repl` | REPL | v9.10.0 |
| `fav watch` | ファイル監視・自動再実行 | v9.9.0 |
| `fav profile` | 実行時間プロファイリング | v9.9.0 |
| `fav explain` | リネージ解析 | v7.1.0 |
| `fav new` | プロジェクト雛形生成 | v10.0.0 |

## 理由

1. **インストールの単純さ**: バイナリ 1 つを PATH に追加するだけで完結する。
   複数バイナリだとバージョン管理が複雑になる。

2. **rvm との相性**: `rvm install 9.12.0` で単一バイナリを差し替えるだけで
   バージョン切り替えが完了する。

3. **サブコマンド間でのコンテキスト共有**: run / check / fmt が同じ
   AST・型情報を共有しやすい。

4. **Go / Rust / Zig の先例**: 主要な現代言語ツールチェーンはほぼ単一バイナリを採用。

## 却下した選択肢

- **複数バイナリ**: バージョン不整合のリスクが高い
  （`fav-check` v9.12 で `fav-run` v9.11 を使うなどの状況が発生しうる）。
- **シェルスクリプトでラップ**: 配布・インストールが複雑になる。

## 影響

- `fav/src/main.rs` がエントリポイントでサブコマンドを dispatch する
- セルフホスト CLI（`fav/self/cli.fav`）も同じサブコマンド体系を実装する
- 新サブコマンド追加時は `cli.fav` と `main.rs` の両方を更新する
