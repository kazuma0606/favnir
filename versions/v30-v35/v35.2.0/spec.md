# v35.2.0 仕様書 — `fav deploy --target docker`

## Background

v35.1.0 では `fav deploy --target lambda` による Lambda 自動デプロイを実現した。
v35.2.0 は Deployment Story スプリントの第二版として、**Docker イメージの自動生成**を追加する。

既存の `cmd_deploy_ecs` / `cmd_deploy_k8s` / `cmd_deploy_fly` は Dockerfile を **書き出すだけ** で
`docker build` を実行しない。v35.2.0 の `--target docker` は **実際に `docker build` を実行する**
点が差別化ポイントである。

また、ECS/K8s/Fly 用の `generate_dockerfile` はインタープリタモード（`fav run pipeline.fav`）向けだが、
v35.2.0 は v35.1.0 と同様に **`fav build --target native` が生成したネイティブバイナリ**を
コンテナに含める専用 Dockerfile テンプレートを新設する。

## Goals

1. `fav deploy --target docker` コマンドを追加する
2. ネイティブバイナリを含む Dockerfile を自動生成する
3. `docker build -t <tag> .` を `std::process::Command` 経由で実行する
4. Docker CLI 不在時は警告を出してスキップする（Lambda の AWS CLI フォールバックと同方式）
5. `--tag` フラグ / `fav.toml [deploy] tag` でイメージタグを指定できる

## Syntax / API

### コマンドライン

```bash
# fav.toml の [deploy] セクションから設定を読み込んで実行
fav deploy --target docker

# タグを明示指定
fav deploy --target docker --tag my-pipeline:latest

# Dockerfile 生成のみ（docker build をスキップ）
fav deploy --target docker --package-only

# --out-dir で出力先を変更（デフォルト: ".fav-deploy/"）
fav deploy --target docker --tag my-pipeline:v1.0 --out-dir ./dist
```

### fav.toml の `[deploy]` セクション

```toml
[deploy]
target = "docker"
tag    = "my-pipeline:latest"
```

### 内部動作フロー

```
1. fav.toml [deploy] または CLI 引数からタグ・out-dir を解決
2. ネイティブバイナリのパスを解決（target/native/<project-name>）
3. generate_dockerfile_native(project_name) で Dockerfile を生成
4. <out-dir>/Dockerfile に書き出す
   （--package-only の場合はここで終了）
5. docker CLI 存在確認（docker --version の exit code）
6. docker build -t <tag> <out-dir>/ を実行
7. 成功/失敗を標準出力に報告
```

## Success Criteria

1. `fav deploy --target docker --package-only` が `<out-dir>/Dockerfile` を生成する
2. 生成された Dockerfile が `debian:bookworm-slim` ベースでネイティブバイナリを COPY する
3. `fav.toml [deploy] tag` が `cmd_deploy_docker` に正しく渡される
4. Docker CLI 不在時に警告を出してスキップし、exit 1 しない
5. `cargo test` が 0 failures で通る（v35200_docker_tests の全テストが pass）
   ※ロードマップでは「Rust テスト 1 件」と記載しているが、網羅性のため 5 件を追加する。
6. `CHANGELOG.md` に `## [35.2.0]` エントリが存在する

## 設計決定

### ロードマップの `fav/src/deploy/docker.rs` について

ロードマップ（`roadmap-v35.1-v36.0.md` v35.2.0 セクション）は `fav/src/deploy/docker.rs` という独立ファイルを成果物として記載しているが、**本バージョンでは `driver.rs` への直接追加を採用する**。

理由: v35.1.0 の `package_lambda` / `deploy_lambda` も `driver.rs` に直接追加しており、独立モジュール化すると両バージョン間で分割ポリシーが不一致になる。`deploy/` モジュールへのリファクタリングは v36.0 クリーンアップ時に一括実施する。

---

### ECS/K8s/Fly との差分

| 既存ターゲット | 新 docker ターゲット |
|---|---|
| Dockerfile を書き出して Next steps を印字 | `docker build` を実際に実行する |
| インタープリタモード（`fav run`） | ネイティブバイナリを直接実行 |
| `generate_dockerfile()` を共用 | `generate_dockerfile_native()` を新設 |

### Dockerfile テンプレート（native 向け）

```dockerfile
FROM debian:bookworm-slim
WORKDIR /app
RUN apt-get update && apt-get install -y ca-certificates && rm -rf /var/lib/apt/lists/*
COPY target/native/<name> /app/pipeline
RUN chmod +x /app/pipeline
ENTRYPOINT ["/app/pipeline"]
# Generated by fav deploy --target docker for project: <name>
```

### `--tag` のデフォルト値

CLI 引数・`fav.toml [deploy] tag` の両方が未指定の場合: `<project-name>:latest`

### `cmd_deploy` シグネチャ変更

v35.1.0 で `package_only: bool` / `output: Option<&str>` を追加した。
v35.2.0 では `tag: Option<&str>` をさらに追加する。

### `versions/current.md` 更新

マイナーバージョン実装では更新しない。

### `mod declare` の置き場所

v35.1.0 の方針を踏襲: `driver.rs` に直接追加（独立モジュールは作らない）。

## Error Codes

新規エラーコードなし。ネイティブバイナリが見つからない場合は標準エラーにメッセージを出力して exit 1。
