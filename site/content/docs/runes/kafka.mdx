# Kafka Rune

`import rune "kafka"` で Apache Kafka（または Redpanda / AWS MSK）に接続できます。`!Stream` エフェクトが必要です。

## セットアップ

```bash
docker run -p 9092:9092 redpandadata/redpanda:latest \
    redpanda start --overprovisioned --node-id 0 \
    --kafka-addr 0.0.0.0:9092 --advertise-kafka-addr localhost:9092
```

Redpanda はローカル開発用の Kafka 互換ブローカーです。AWS MSK / Confluent Cloud にも同じ API で接続できます。

> **接続モデル（v25.7.0）**: 各 `Kafka.connect` 呼び出しのたびに `list_topics` ping を送信します。コネクションプールは v26.x で対応予定。

環境変数:
- `KAFKA_BOOTSTRAP_BROKERS` — カンマ区切りのブローカーアドレス（例: `localhost:9092`）
- `KAFKA_SASL_USERNAME` / `KAFKA_SASL_PASSWORD` — SASL/SCRAM-SHA-512 認証（省略可）

## `!Stream` エフェクト

`!Stream` は外部 Kafka/ストリーミング操作を明示するエフェクトです。宣言がない場合は **E0319** エラーが発生します。

```favnir
stage PublishEvent: String -> Result<Unit, String> = |event_json| {
    bind conn <- Kafka.connect("localhost:9092")
    Kafka.produce(conn, "events", "event-key", event_json)
}
```

## API リファレンス

### `Kafka.connect`

```favnir
fn connect(brokers: String) -> Result<KafkaConn, String>
```

ブローカーへの接続確認（`list_topics` ping）を行い、`KafkaConn` を返します。

- `""` → `KAFKA_BOOTSTRAP_BROKERS` 環境変数 → `"localhost:9092"`
- それ以外 → 指定文字列をそのまま使用

```favnir
bind conn <- Kafka.connect("localhost:9092")
```

---

### `Kafka.produce`

```favnir
fn produce(conn: KafkaConn, topic: String, key: String, value: String) -> Result<Unit, String>
```

指定トピックにメッセージを produce します（partition 0 固定）。

```favnir
bind _ <- Kafka.produce(conn, "events", "user-123", "{\"type\": \"signup\"}")
```

---

### `Kafka.consume_one`

```favnir
fn consume_one(conn: KafkaConn, topic: String, group_id: String) -> Result<String, String>
```

トピックの最新オフセットから 1 件取得します。

- トピックが空の場合: `Result.err("Kafka.consume_one_raw: topic is empty")`

> **注意**: `group_id` は rskafka v0.6 の制約により現時点では未使用です（v26.x で Consumer Group 実装予定）。

```favnir
bind msg <- Kafka.consume_one(conn, "events", "my-group")
```

---

### `Kafka.consume_batch`

```favnir
fn consume_batch(conn: KafkaConn, topic: String, group_id: String, max_count: Int) -> Result<String, String>
```

最新オフセットから最大 `max_count` 件のメッセージ payload を JSON 配列文字列で返します。

- トピックが空の場合: `Result.ok("[]")`
- 戻り値例: `"[\"msg1\", \"msg2\", \"msg3\"]"`

> **注意**: `group_id` は rskafka v0.6 の制約により現時点では未使用です（v26.x で Consumer Group 実装予定）。

```favnir
bind msgs_json <- Kafka.consume_batch(conn, "events", "etl-group", 10)
```

---

### `Kafka.create_topic`

```favnir
fn create_topic(conn: KafkaConn, topic: String, partitions: Int) -> Result<Unit, String>
```

トピックを作成します（partition 数指定、replication factor 1）。

- トピックが既に存在する場合はエラーを返します。

```favnir
bind _ <- Kafka.create_topic(conn, "events", 3)
```

---

## VM プリミティブ（低レベル API）

| プリミティブ | 引数 | 戻り値 |
|---|---|---|
| `Kafka.connect_raw` | `brokers: String` | `Result<String, String>` |
| `Kafka.produce_raw` | `brokers, topic, key, value: String` | `Result<Unit, String>` |
| `Kafka.consume_one_raw` | `brokers, topic, group_id: String` | `Result<String, String>` |
| `Kafka.consume_batch_raw` | `brokers, topic: String, max_count: Int` | `Result<String, String>`（JSON 配列） |
| `Kafka.create_topic_raw` | `brokers, topic: String, partitions: Int` | `Result<Unit, String>` |

## スコープ外（v25.7.0）

- Consumer Group オフセット管理（`commit` / `seek`）— rskafka v0.6 は Consumer Group プロトコル未対応
- スキーマレジストリ（Avro / Protobuf デシリアライズ）
- Exactly-Once Semantics（EOS / transactional producer）
- 複数 partition のラウンドロビン送信（partition 0 固定）

## 使用例

```favnir
import rune "kafka"

stage PublishEvent: String -> Result<Unit, String> = |event_json| {
    bind conn <- Kafka.connect("localhost:9092")
    bind _    <- Kafka.create_topic(conn, "events", 1)
    Kafka.produce(conn, "events", "event-key", event_json)
}

stage ConsumeEvents: Unit -> Result<String, String> = |_| {
    bind conn <- Kafka.connect("localhost:9092")
    Kafka.consume_batch(conn, "events", "etl-group", 10)
}

seq EventsETL = PublishEvent |> ConsumeEvents
```

## AWS MSK との接続

SASL/SCRAM-SHA-512 認証:

```bash
export KAFKA_BOOTSTRAP_BROKERS="b-1.msk.ap-northeast-1.amazonaws.com:9096,b-2.msk.ap-northeast-1.amazonaws.com:9096"
export KAFKA_SASL_USERNAME="your-username"
export KAFKA_SASL_PASSWORD="your-password"
```
